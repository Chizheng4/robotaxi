# v034 经营指标系统版本计划

## 1. 版本定位

- 版本：`v034`
- 类型：大版本
- 当前阶段：方案与执行计划
- 自动推进：否。本文档完成后，需用户确认再进入代码实现。

v034 的目标是建立最小经营指标闭环，让模拟运行产生的业务数据、成本数据和收入数据能够被计算、展示、解释和下钻。

## 2. 子版本拆分

### v034.0 指标系统方案与字段合同

范围：

1. 完成 06 经营指标系统方案。
2. 明确指标对象模型：MetricDefinition、MetricCalculationRun、MetricObservation。
3. 明确第一版指标目录和 READY / DERIVABLE / MISSING_DATA 就绪度。
4. 明确前端体验结构和经营分析菜单调整。

交付：文档，不编码。

### v034.1 指标定义与计算引擎

范围：

1. 新增指标定义初始化。
2. 新增指标计算服务。
3. 支持按 SimulationRun 全局窗口计算 P0 指标。
4. 生成 MetricCalculationRun 和 MetricObservation。
5. 成本或收入未完成时输出质量状态，不静默计算错误利润。

### v034.2 经营指标总览页面

范围：

1. 在经营分析管理中新增“经营指标总览”。
2. 展示核心指标：实收收入、总成本、贡献利润、订单履约率、单均成本。
3. 支持指标详情右侧面板，展示定义、公式、来源字段、质量状态。
4. 保持统一前端骨架、字段字典和轻量视觉风格。

### v034.3 财务与服务指标页面

范围：

1. 新增财务指标页面：收入、成本、利润、利润率、单均收入、单均成本。
2. 新增服务指标页面：创建订单、完成订单、取消订单、履约率、服务覆盖。
3. 支持按模拟运行、模拟日、服务区、车辆等维度扩展。

### v034.4 过程诊断与质量护栏

范围：

1. 新增过程诊断页面：匹配、路径、履约、任务完成。
2. 新增指标质量状态：PASS、WARN、FAIL。
3. 指标详情支持来源对象下钻。
4. 校验成本、收入、模拟时间、状态时间线完整性。

### v034.5 自动计算与验证闭环

范围：

1. 模拟运行完成、成本计算完成、收入生成完成后自动触发指标计算。
2. 支持人工重新计算指标。
3. 更新 `VERSION.md`、字段字典、前端字段展示合同。
4. 增加专项验证脚本和页面加载验证。

## 3. 第一版 P0 指标

|指标|来源|页面|
|---|---|---|
|应收收入|RevenueRecord|经营指标总览、财务指标|
|实收收入|RevenueRecord|经营指标总览、财务指标|
|未收收入|RevenueRecord|财务指标|
|运营总成本|CostRecord|经营指标总览、财务指标|
|贡献利润|RevenueRecord - CostRecord|经营指标总览、财务指标|
|贡献利润率|贡献利润 / 实收收入|财务指标|
|创建订单数|ServiceOrder|服务指标|
|完成订单数|ServiceOrder|经营指标总览、服务指标|
|订单履约率|完成订单数 / 创建订单数|经营指标总览、服务指标|
|订单取消率|取消订单数 / 创建订单数|服务指标|
|单均收入|实收收入 / 完成订单数|财务指标|
|单均成本|运营总成本 / 完成订单数|经营指标总览、财务指标|
|关键数据完整率|模拟时间、成本、收入、状态时间线|质量护栏|

## 4. 前端体验要求

1. 不做营销式大屏，不使用大标题、厚重色块或装饰背景。
2. 指标总览必须信息密度高但层次清晰。
3. 指标卡片用于表达单个核心指标，避免卡片套卡片。
4. 趋势、构成、异常和详情必须能互相解释。
5. 所有指标名称、字段名、状态和枚举必须来自统一字段字典。
6. 指标详情必须展示：定义、公式、来源字段、窗口、质量状态、来源对象。

## 5. 暂停条件

进入编码后如出现以下情况必须暂停确认：

1. 指标字段命名存在多种合理选择。
2. 成本或收入事实与业务单据对账不一致。
3. 指标页面需要新增超出经营分析管理的导航结构。
4. 自动计算触发会造成性能风险。
5. 需要新增车辆状态历史快照、潜在需求事实等 P1 能力。
