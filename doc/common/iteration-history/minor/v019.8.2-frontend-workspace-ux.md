# v019.8.2 前端工作台体验优化

## 背景

随着运营对象、单据和管理页面持续增加，运营人员在执行完整业务闭环时需要频繁在多个菜单页面之间切换。

当前模式下，需要不断进入菜单、执行操作、切换菜单、返回原页面，操作成本逐步升高。

## 目标

提升运营闭环测试与日常运营效率。

支持多个业务页面并行保留与快速切换。

减少重复进入菜单的操作成本。

## 约束

- 保持当前整体布局结构。
- 保持当前字体、颜色和视觉风格。
- 保持左侧菜单导航模式。
- 保持右侧 Detail 面板模式。
- 中间业务区域仍然是页面核心。

## 实现方案

采用紧凑工作台页签条，而不是传统大面积 Tab。

原因：

- 当前系统左侧菜单仍然是主导航；
- 工作台只负责保留和快速切换已打开页面；
- 页签条高度必须很低，避免挤压中间业务区域；
- 每个页面状态独立保存，避免跨页面筛选、分页和详情状态互相干扰。

本轮页面状态结构：

- 已打开页面列表；
- 当前激活页面；
- 页面筛选条件；
- 页面已应用筛选条件；
- 当前分页；
- 当前选中记录；
- 当前页面 Detail 展开 / 收缩状态。

## 体验优化

- 页签区域只保留一行，不允许换行挤压中间业务区；
- 页签之间保留克制间距，视觉上自然区分；
- 页签选中态沿用系统主色和浅蓝背景，不使用突兀大色块；
- 页签区域不能形成额外突兀分割层，左侧菜单、中间业务区、右侧详情区仍以现有细线分隔为主；
- 中间业务区与右侧详情区顶部、底部对齐；
- 表格区域、页签区域、详情区域高度计算统一，避免上下边界不平行；
- 工作台页签服务快速切换，不抢占业务内容注意力。

## 验证结果

- 提交前检查通过；
- 工作台页签可新增、切换、关闭；
- 页面筛选、分页、选中记录和详情展开状态可按页面保留；
- 刷新后工作台状态可恢复；
- 页签保持单行；
- 中间业务区与右侧详情区顶部、底部对齐；
- 浏览器控制台无前端错误。
