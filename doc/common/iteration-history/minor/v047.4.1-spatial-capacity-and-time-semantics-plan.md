# v047.4.1 空间承载与时间语义迭代计划

## 目标

统一 ServiceArea、Zone 与长期需求预测的承载计算，固定订单时间字段边界，并让字段名称、业务定义和计算公式由统一语义服务提供。

## 任务

- [x] 固定 Place 产生需求、ServiceArea 提供承载、Zone 汇总需求与承载的边界。
- [x] ServiceArea 分别计算上车与下车能力，Zone 汇总后识别端到端瓶颈。
- [x] 固定载客行驶、订单履约执行、订单全流程和车辆服务周期四个时间概念。
- [x] 扩充字段语义注册与计算模型注册，计算结果只保存字段键、模型编号、输入、来源和版本。
- [x] 迁移长期预测到统一字段与模型，兼容读取旧快照但不继续写旧字段。
- [x] 统一前端“生产供应”和相关供应字段名称，补齐画像与预测计算解释。
- [x] 增加领域、迁移、字典、桌面和手机端验证。

## 验收

- [x] ServiceArea 不生成需求，保存配置后 Place、ServiceArea、Zone 使用同一重算服务。
- [x] Zone 日承载与峰值承载均按汇总后的上车、下车能力最小值计算。
- [x] 预测中的车辆产能只使用车辆服务周期，四个时间字段不存在概念混用。
- [x] 修改统一字段中文名后，表格、详情和计算过程同步变化。
- [x] 构建检查可阻止未登记字段、模型和旧中文口径进入前端。
- [x] 提交前检查、真实桌面和手机页面验证通过。
