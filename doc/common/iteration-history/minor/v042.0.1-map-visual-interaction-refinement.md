# v042.0.1 地图视觉与交互收敛

## 目标

在不改变空间对象、业务闭环和模拟运行的前提下，将双区域地图收敛为轻量、精致、可识别的运营地图，并修复车辆和 Cell 交互缺口。

## 完成内容

1. 顶部标题统一为“地图空间”，移除冗余空间统计摘要。
2. Zone、地点、服务区域和道路统一使用锚点式轻量标签，标签不随地图缩放变大。
3. 缩放层级切换区域、地点、服务区域和道路标签，避免信息同时堆叠。
4. 道路恢复为 RoadSegment 覆盖 Cell 的合并面。
5. 地图扩展到完整工作区底层，详情作为上层对象检查器。
6. 修复车辆集合入口，所有存在当前位置的 Robotaxi 均生成地图点位。
7. Cell 点击显示唯一选中框，并清除旧悬浮信息。
8. 悬浮摘要减少为对象类型和最多两项关键指标。
9. README 当前更新纳入版本提交。

## 验证

- 多区域空间、Zone 1 业务隔离和旧快照迁移合同通过。
- 真实浏览器确认 20 个 Robotaxi 点位全部生成。
- 真实浏览器确认 Cell 点击后存在唯一选中框且悬浮提示关闭。
- 真实浏览器确认地图节点低于 500、桌面和手机无全局横向溢出。
- 完整提交前检查通过。
