# v046.0.7 决策监控统计与数据合同加固

## 目标

决策中心继续消费各价值流唯一业务事实，不新增重复事实对象；修正订单匹配重试被误解为大量受影响订单的问题，并建立可追溯的数据版本和中文展示合同。

## 任务

- [x] 将失败执行按异常类别归一化，但保留源执行状态和原因。
- [x] 按“决策能力 + 调用业务对象”聚合决策过程。
- [x] 同时展示异常尝试数、异常过程数、受影响对象数、恢复数和终局影响数。
- [x] 增加来源版本、来源更新时间、指标更新时间和待重算提示。
- [x] 验证重置后订单匹配事实及决策投影归零。
- [x] 未登记内部枚举不得直接透传到页面。
- [x] 将经营规划的预测结果提升为二级菜单，不改变预测服务和结果对象。
- [x] 完成服务测试、字段合同、导航检查和桌面/手机真实页面验证。

## 验收

1. 同一服务订单连续 100 次失败显示为 1 个受影响订单、100 次异常尝试。
2. 后续匹配成功时显示重试后恢复，不再列为待处理终局异常。
3. 重置后源事实、决策过程、异常和页面刷新结果全部归零。
4. 决策中心不显示未登记的内部英文枚举。
5. 预测结果可从经营规划二级菜单直接进入。
