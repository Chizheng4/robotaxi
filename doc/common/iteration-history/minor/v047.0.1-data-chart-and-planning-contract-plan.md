# v047.0.1 数据图表与经营规划合同迭代计划

## 目标

统一经营规划与经营分析的数据图表基础设施，修正经营目标、需求预测与计划承接量的口径，并让完整计算过程严格绑定字段字典和来源数据。

## 任务

- [x] 经营目标、市场预测、计划承接量和履约质量职责分离；
- [x] 完整计算过程保存输入字段、计算模型、公式、输出字段和来源；
- [x] 所有计算结果名称由统一字段字典取得，不在页面维护别名；
- [x] 共享图表控件改用本地 Apache ECharts 精简发行文件；
- [x] 图表统一支持响应式尺寸、准确浮动提示、长序列浏览和手机触摸；
- [x] 正常供应决策与自动演示供应流程分离；
- [x] 同步正式方案、字段字典、前端规则、自动验证和版本记录；
- [x] 完成桌面端、手机端、性能与真实页面验收。

## 验收

1. 图表指示线、数据点和浮动提示使用同一坐标体系；
2. 稀疏数据铺满可用空间，长序列在图内缩放或浏览，不产生页面横向溢出；
3. 计算步骤可连续解释 Place 到 Zone、增长、计划承接、Robotaxi 能力、缺口和供给；
4. 页面不再显示“当前需求基线”“经营目标比较”“峰值并发”等非正式输出名称；
5. 手机端可阅读指标、图表和计算过程，交互不遮挡页面；
6. 业务服务、模拟运行和既有经营数据刷新边界不变。

## 完成结果

- 需求预测口径、字段字典、计算步骤、正式方案和供应动作边界已统一；
- 经营规划与经营分析通过共享 ECharts 控件使用同一图表交付方式；
- 合同、性能、桌面与手机真实页面验证通过。
