# v047.5.0 决策可解释架构迭代计划

## 目标

统一策略、执行、结果的对象性质、计算过程数据合同和详情控件，使用户能够用一致方式理解决策使用的数据、模型、公式、结果和下游影响，并通过代码合同阻止后续页面继续自建个性化结构。

## 任务

- [x] 更新决策控制与字段语义正式设计，固定五类结果性质及计算过程责任边界。
- [x] 扩展决策能力目录，声明结果性质、计算过程来源、生命周期和下游对象。
- [x] 建立统一计算过程解析与校验服务，兼容历史字段但限制新增结构。
- [x] 建立共享计算过程详情控件，统一画像、预测、供应决策和生产计划的展示。
- [x] 交叉校验决策能力目录与页面架构注册，防止结果性质和页面类型漂移。
- [x] 增加字段中文、模型、公式、单位和来源合同验证。
- [x] 完成桌面、手机和真实页面加载验证。
- [x] 更新版本、站内记录、归档计划、提交并创建标签。

## 验收

1. 十二项现有决策能力全部声明结果性质和解释合同。
2. 画像、预测、供应决策执行和生产计划复用同一计算过程控件。
3. 公式、输入和输出不直接显示内部字段键或内部单位枚举。
4. 决策目录与页面架构不一致时验证失败。
5. 历史快照正常展示，业务服务和模拟运行路径不受影响。
