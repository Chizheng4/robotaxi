# v042.0.7 地图真实命中与 Cell 选择

## 问题根源

1. SVG 在 `pointerdown` 时立即捕获指针，真实点击 Robotaxi 后的事件被重定向到地图画布。
2. 历史 CSS 将服务区域、运营中心和道路节点设置为不可命中，后续视觉覆盖未恢复交互。
3. 浏览器测试直接向 DOM 派发事件，绕过了真实图层命中，形成错误通过。
4. 空白点击逻辑取消了 Cell 选择，导致选中视觉消失。

## 完成内容

1. 只有拖动超过阈值后地图才捕获指针，点击和拖动边界分离。
2. 最终地图样式恢复服务区域、运营中心和道路节点命中能力。
3. 空间服务按 RoadNode、ServiceArea、OpsCenter、Place、RoadSegment 顺序解析 Cell 对象归属。
4. 空白 Cell 显示选择框但保持详情收起；地图边界外留白清除选择。
5. 浏览器合同使用 CDP 真实鼠标和触摸输入，并用 `elementFromPoint` 检查实际命中层。

## 验证

- 1440 × 900 桌面真实鼠标交互通过。
- 390 × 844 手机真实触摸交互通过。
- 地点、道路、服务区域、Robotaxi 和 Cell 均通过真实命中合同。
- 10,000 辆 Robotaxi 投影约 2.34ms；100 次双区域场景生成约 71.92ms，静态节点 53。
- 本轮未新增字段、状态或枚举，无需修改统一字段字典。
