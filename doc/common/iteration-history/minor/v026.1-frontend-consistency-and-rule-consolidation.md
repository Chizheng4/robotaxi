# v026.1 前端视觉一致性与规则集中化

## 迭代类型

小迭代。

## 目标

1. 去除品牌和左右收展控件不一致的玻璃状悬浮反馈。
2. 建立清晰的一级、二级、三级菜单层级。
3. 提高选中、悬停和父级状态的辨识度。
4. 提升中文字体清晰度，同时保持克制轻盈。
5. 将前端体验与设计系统规则集中到单一权威文件。
6. 让 `AGENTS.md` 成为唯一任务分流入口，避免其他模型漏读或误解。

## 实施内容

- 为菜单建立 `14px / 34px / 54px` 的三级递进缩进和稳定字重。
- 加深选中与悬停背景、文字颜色和父级激活状态，选中项增加左侧标记。
- 调整字体渲染及正文、筛选、表格和表头颜色；将大部分业务文字保持在 `400 / 500`，只让关键标题和选中项使用 `600`。
- 去除品牌和左右收展按钮的玻璃状悬停样式及原生 `title` 气泡，补充可访问名称。
- 将 `07-frontend-design-system.md` 的稳定内容合并到 `04-frontend-ux-rules.md`，并删除活动规则 07。
- 在统一规则中明确视觉艺术家标准：以比例、留白、细边界、对齐和克制字重实现精致轻盈，禁止用粗大文字和厚重色块制造清晰度。
- 更新 `AGENTS.md`、规则索引与迭代入口，消除重复和冲突的强制读取清单。
- 更新前端缓存版本、重新生成 bundle，并补充空 favicon 避免无意义资源错误。

## 字段字典

本轮未新增或修改业务对象、字段、状态或枚举，因此未修改字段字典。

## 验证结果

- `src/main.bundle.js` 已由 `src/main.jsx` 重新生成并通过语法检查。
- 真实页面实测三级菜单缩进为 `14px / 34px / 54px`，各层字重为 `500 / 400 / 400`，选中项为 `600`。
- 选中态与悬停态颜色明确区分，父级激活层级清晰。
- 左侧导航可在 `60px / 232px` 间收展；右侧详情可隐藏并恢复。
- 品牌和收展控件无原生 `title` 气泡，无玻璃状悬停背景。
- 1280 x 720 与 1440 x 900 均无页面级溢出、重叠或遮挡。
- 页面请求无 4xx/5xx，控制台无新增错误或警告。
- 提交前项目检查通过。
