# v042.0.4 地图对象交互恢复

## 问题根源

1. 地图为了在选择后清除旧悬浮摘要，错误地在存在任何选中对象时禁止了后续全部悬浮。
2. Zone 的内部填充仍可能参与命中，导致内部地点、道路、服务区域和 Robotaxi 的点击边界不稳定。

## 完成内容

1. 点击对象只清除当下悬浮摘要，不禁止后续对象重新触发悬浮。
2. Zone 选择只响应边界线，内部运营对象保持独立交互权。
3. Robotaxi 使用统一地图选择入口打开详情，不在页面复制详情逻辑。
4. 浏览器合同覆盖 Cell 选择、地点悬浮、Robotaxi 点击和唯一车辆详情切换。

## 验证

- 1440 × 900 真实浏览器完整交互链通过。
- 地图仍生成 20 个存在当前位置的 Robotaxi 图标。
- 交互恢复未改变空间对象、业务服务或模拟运行。
