# v040.17 Robotaxi 顶部体验与任务规划排队序号修复

## 背景

用户基于 Robotaxi 管理页截图反馈：

1. 顶部体验一致化较差，当前选中 Robotaxi 像独立卡片，和状态分类、筛选、表格层级关系不清晰。
2. 字段展示不全，任务规划执行和结果无法清楚解释排队内容。
3. Robotaxi 排队任务仍没有任务序号，怀疑任务规划策略执行结果本身没有生成。
4. 前端体验优化必须有明确方案和验证内容，不能只做临时样式。

## 根因

- v040.16 只验证了存在 `formatRobotaxiQueueItems`，没有验证任务规划结果数据模型中存在排队序号。
- 任务规划结果只保存 `queue_entry`，其中仅包含任务类型和优先级，缺少可复盘的 `queue_sequence`。
- Robotaxi 顶部区域左侧指标与右侧当前车辆信息使用了不同视觉结构，右侧 `robotaxi-focus-block` 形成了独立卡片感。
- Robotaxi 详情的任务状态分组未纳入 `pending_task_queue`，导致表格选中车辆后右侧详情仍看不到完整队列。
- 任务规划执行 / 结果列表字段过窄，未暴露输出快照、队列项和排队序号。

## 变更

- `src/services/robotaxiTaskPlanningService.js`
  - 任务规划进入队列时生成 `queue_sequence`。
  - 同优先级任务排在已有同优先级任务之后，序号按“高优先级 + 同优先级已有任务 + 1”计算。

- `src/domain/robotaxiTaskPlanningTypes.js`
  - `RobotaxiTaskPlanningResult` 增加顶层 `queue_sequence`，优先来自结果字段，兼容读取 `queue_entry.queue_sequence`。

- `src/main.jsx`
  - Robotaxi 顶部区域改为统一概览结构：当前选中、当前位置、当前占用、运维状态、排队任务、当前可触发运维。
  - 移除旧 `robotaxi-focus-block` 结构，避免当前选中车辆像单独卡片。
  - 队列展示优先读取 `queue_sequence`，兼容旧数据按位置兜底，并展示任务编号和优先级。
  - Robotaxi 详情任务状态分组新增 `pending_task_queue`、`pending_fleet_task_type`、`pending_fleet_task_id`。
  - 任务规划结果列表新增 `queue_sequence` 和 `queue_entry`；任务规划执行列表新增 `output_snapshot`。

- `src/styles.css`
  - Robotaxi 顶部信息改为统一细边界键值布局，避免标签截断和卡片套卡片。

- `src/domain/fieldDictionary.js`、`doc/rules/field-dictionary.md`
  - 新增 `queue_sequence / 排队序号`。

- `scripts/verify-field-display-contract.mjs`
  - 字段展示合同样本覆盖任务规划队列序号。

- `scripts/verify-v040-17-robotaxi-planning-queue-ux.mjs`
  - 新增提交前合同，覆盖顶部结构、旧卡片块移除、排队序号模型、字段字典和字段展示合同。

- `scripts/verify-v040-17-robotaxi-layout-browser.mjs`
  - 新增真实浏览器布局验证脚本，按菜单进入 Robotaxi 管理页并检查 1280×720、1440×900 下关键标签不截断。

## 边界

- 本轮不改变模拟运行主路径。
- 本轮不改变任务规划策略的优先级算法，只补充排队序号可解释性和前端展示一致性。
- 浏览器布局脚本不纳入提交前自动检查，避免提交检查依赖本机 Chrome；本轮已手动运行两个尺寸。
