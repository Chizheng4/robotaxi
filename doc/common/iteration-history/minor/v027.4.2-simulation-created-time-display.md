# v027.4.2 模拟业务双创建时间展示

## 1. 迭代目标

为模拟运营分析需要的关键业务对象完整展示真实创建时间与模拟创建时间，并保证数据写入、字段字典、表格和详情形成一致闭环。

## 2. 统一定义

- `created_at` / 创建时间：对象写入系统时的真实审计时间。
- `simulation_created_at` / 模拟创建时间：对象在连续模拟时间轴中生成时的 `Day N HH:MM:SS`。
- 人工创建或审计字段上线前的历史记录，模拟创建时间为空，不使用真实时间倒填。

## 3. 完成范围

1. 运营准入任务、运营投放任务和服务订单显示双创建时间。
2. 执行引擎生成的运营行驶记录和履约行驶记录显示双创建时间。
3. 五个表格统一按“创建时间 → 模拟创建时间”排列。
4. 五类对象详情统一设置“时间与来源”，同时展示模拟运行、记录来源和全局 Tick。
5. 运营行驶记录补齐真实创建时间；运营准入任务改用标准分栏详情。

## 4. 字段字典检查

本轮没有新增或修改业务字段、状态和枚举。`created_at` 与 `simulation_created_at` 已同时存在于：

- `doc/rules/field-dictionary.md`
- `src/domain/fieldDictionary.js`

字段定义和中文名称保持一致，不新增重复命名。

## 5. 自动验证

- 模拟 handler 验证五类对象都写入真实创建时间和模拟创建时间。
- 页面配置验证五个表格和五类详情中双时间相邻展示。
- `src/main.bundle.js` 与 `src/main.jsx` 一致并通过语法检查。
- 提交前完整检查通过。
