# v033.1 小迭代：时间作业诊断与清理

## 版本类型

小迭代。

## 目标

把模拟事件与时间作业两类诊断数据分开管理，避免用户在错误页面执行不对应的清理动作；同时让重置模拟数据能够真正清空时间作业，减少无效运行态数据对后续模拟的影响。

## 问题识别

1. `重置模拟数据` 清空了模拟事件与主要业务运行态，但没有明确清空内存中的时间作业。
2. 最近事件与时间作业的检查意义不同，不应该合并为一个清空入口。
3. 时间作业页面缺少业务单类型过滤，排查某类业务单据的调度正确性不够聚焦。
4. 清空全部时间作业如果发生在运行中，会破坏统一时间世界下的调度队列。

## 执行结果

- 新增 `src/data/timedOperationDiagnostics.js`，把时间作业终态清理、活跃运行保护、业务单类型过滤抽成纯函数。
- `重置模拟数据` 同步清空内存时间作业。
- 时间作业页面增加 `业务单类型` 筛选，复用统一字段字典中文展示。
- 时间作业页面增加 `清空已结束作业` 与受保护的 `清空全部作业`；模拟运行仍在 `RUNNING / DRAINING / PAUSED` 时不允许清空全部作业。
- 模拟事件清空继续只放在模拟运行上下文，保持事件诊断与时间作业诊断分离。
- 字段字典补齐 `object_type` 枚举中文，避免时间作业筛选暴露内部英文值。
- 新增 `scripts/verify-v033-1-timed-operation-diagnostics.mjs` 并接入提交前检查。
