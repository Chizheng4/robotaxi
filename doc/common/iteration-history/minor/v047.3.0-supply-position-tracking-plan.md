# v047.3.0 供给跟踪迭代计划

## 目标

基于生产计划、生产批次、Robotaxi、交付编排、交付单和运营准入事实，形成不重复计数、可追溯、自动更新的供给跟踪视图。

## 任务

- [x] 固定供给跟踪的对象边界、三种分析口径和阶段互斥规则。
- [x] 实现供给跟踪投影服务，按来源事实生成区域汇总、阶段记录和数据诊断。
- [x] 将供给跟踪接入供应管理，使用分析型页面且不增加最近事件或独立持久化事实。
- [x] 同步字段字典、中文枚举、导航合同和前端响应式样式。
- [x] 验证数量守恒、Robotaxi 不重复计数、来源追溯、桌面与手机页面。
- [x] 更新版本记录、归档计划、提交并创建版本标签。

## 验收

1. 当前可用供给、区域资产、已承诺在途和未来供给管道口径彼此清晰。
2. 每辆 Robotaxi 最多出现在一个供给阶段；批次数量和已形成车辆资产不会重复相加。
3. 每条跟踪记录能够追溯到生产计划、生产批次、交付编排结果、交付单或 Robotaxi。
4. 页面随底层业务事实变化自动更新，不创建第二份供给业务事实。
5. 页面、字段、枚举全部中文，并通过桌面与手机真实加载验证。
