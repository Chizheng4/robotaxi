# v045.1.0 全平台页面展示控件标准化

## 目标

通过公共页面展示合同覆盖全平台地图、分析和记录页面，使响应式、横向浏览、详情页签和行操作行为由控件继承，不再逐页适配。

## 完成任务

- [x] 盘点全部页面入口，确认业务页统一经过 `RecordTable` 和 `DetailPanel`。
- [x] 在页面上下文服务中建立地图、分析和记录三类展示合同。
- [x] 为经营规划、经营模型和经营分析统一接入分析内容视口。
- [x] 统一数据与指标页面的手机重排和组件内部横向浏览边界。
- [x] 修复行内功能触发行选中及详情展开的问题。
- [x] 统一详情页签溢出和手机详情收起行为。
- [x] 更新前端架构规则并新增页面展示合同自动检查。
- [x] 完成桌面列表操作、手机经营总览和手机多页签详情真实浏览器验证。

## 架构边界

- 页面只提供业务数据、字段、动作和必要插槽。
- 页面展示类型由 `pageContextService.resolvePagePresentation` 统一决定。
- 分析页面使用 `AnalysisContentViewport`；记录页面使用统一表格、操作列和详情栏。
- 行操作由 `RowActionButton`、`RowActionGroup` 和操作单元格统一隔离事件。
- 本版本不修改业务对象、状态、字段、数据计算和模拟运行闭环。
