# v041.2.16 紧凑导航与顶部工具结构优化

## 目标

根据现有菜单层级和文案长度减少左侧导航无效留白，并重新组织顶部版本、用户和退出关系，使全局工作台更紧凑、清晰且字体一致。

## 边界

1. 只调整全局导航宽度、平台工具结构和统一样式，不修改菜单层级及业务页面。
2. 迭代记录继续消费 `VERSION.md` 生成数据，账户菜单只负责退出当前访问会话。
3. 不改变业务运行态、业务服务、模拟运行、表格、详情和事件区域。

## 完成任务

- [x] 任务 1：左侧导航由 232px 收敛为 200px，并同步手机展开宽度。
- [x] 任务 2：顶部工具重新分为页面指标、版本入口和账户入口。
- [x] 任务 3：迭代记录入口改为显示最新版本号，保留原浮层能力。
- [x] 任务 4：将退出操作收纳到“金星”账户展开菜单，减少常驻空间占用。
- [x] 任务 5：统一顶部标签、按钮、版本号和用户入口为 13px、28px 高。
- [x] 任务 6：重建 bundle，完成桌面、手机、浮层交互和工程合同验证。

## 验证结果

- `1280 x 720` 下导航宽度为 200px，三级菜单仍可完整显示，主内容获得更多横向空间。
- 顶部记录、重置、版本号和登录用户字体均为 13px，控件高度均为 28px。
- 点击版本号显示迭代记录；点击“金星⌄”显示退出，两类浮层不会同时占用顶部空间。
- 业务页面结构、表格、详情和模拟运行合同未发生变化。
