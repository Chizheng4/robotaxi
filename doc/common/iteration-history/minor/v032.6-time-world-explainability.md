# v032.6 时间世界解释性与未完成版本补完

状态：已完成。

类型：小迭代补完。

## 背景

`v032` 大版本执行计划中包含 `v032.6 运行失败定位、验证合同、归档提交`，但此前只提交到 `v032.5` 并提前归档。用户同时指出当前前端仍无法清晰判断统一模拟世界时间、时间作业耗时来源和路径信息是否正确。

## 修复目标

1. 补完 `v032.6` 并提交标签。
2. 默认模拟 Tick 改为秒级推进。
3. 运行时自动化动作耗时统一从 `execution_time_config` 读取。
4. 时间作业展示改为“业务单类型 / 业务单编号”，并展示开始秒、计划完成秒、计划完成模拟时间。
5. 修复服务订单缺少需求模拟结果编号。
6. 路径信息详情改为用户友好的关联路径摘要。
7. 增加验证合同，证明时间作业按配置秒数和统一模拟秒计算。

## 关键实现

- `SimulationPolicy` 默认 `tick_seconds = 1`，`tick_minutes = 1 / 60` 仅保留兼容表达。
- `DemandSimulationRun` 生成 `demand_simulation_result_id`，服务订单创建时保存该编号。
- `TimedOperation` 创建时写入 `simulation_planned_completed_at`。
- 行驶时间作业使用 `execution_time_config.cell_travel_seconds`，进度投影与完成动作使用同一配置来源。
- 运营准入使用 `execution_time_config.readiness_check_seconds`。
- 时间作业列表增加 `start_seconds / planned_completed_seconds`。
- 模拟运行详情增加“统一模拟时间说明”。
- Trip / RouteExecution 路径信息使用 `route_links_detail` 专用摘要展示。

## 验证

- `scripts/check-before-commit.sh`
- `scripts/verify-v032-6-time-explainability.mjs`
- 本地页面入口 HTTP 验证。

## 后续

`WorkflowTimingProfile` 继续用于模拟完成后的状态时间线计算；`execution_time_config` 用于模拟运行中的真实时间作业耗时。两者不是同一配置来源，后续前端可进一步增加说明文案或配置编辑体验。
