# v040.16 Robotaxi 管理层级与运维行驶闭环修复

## 背景

用户反馈七类问题：

1. Robotaxi 管理页的运营状态面板仍在状态分类和筛选之下，卡片式表达与表格筛选层级不清晰。
2. 任务规划策略、执行、结果和 Robotaxi 详情仍有英文字段或枚举，说明字段展示合同没有覆盖嵌套快照。
3. Robotaxi 排队任务缺少排队序号，“可操作任务 暂无”对用户含义不清。
4. 清洁等运维任务详情没有展示 Robotaxi 当前位置信息。
5. 运维任务进入待行驶状态后仍需要人工点击生成行驶记录，不符合已确认的自动闭环。
6. 待行驶状态应查看统一运营行驶记录，而不是在任务单里私有化创建。
7. 运营行驶记录页面缺少路径规划、继续行驶、正常到达、异常到达等业务闭环动作入口。

## 根因

- Robotaxi 管理页新增的运营面板被放在通用列表工具区之后，导致“运营总览”和“筛选明细”层级倒置。
- 字段展示合同样本未覆盖任务规划策略执行和结果中的 `strategy_snapshot`、`input_snapshot`、`output_snapshot`、`composite_state` 等嵌套结构，英文键和值没有被统一校验发现。
- 运维任务详情仍沿用普通详情字段集合，没有按清洁、充电、维修、故障、退役任务的闭环字段分组展示位置和行驶记录。
- 运维任务分配目的站点后没有立即调用统一运营行驶记录创建能力，导致页面出现“生成行驶记录”的补偿动作。
- 运营行驶记录行操作被前序迭代覆盖，缺少独立页面的行驶推进入口。

## 变更

- `src/main.jsx`
  - Robotaxi 管理页运营状态面板上移到状态分类和筛选之前，形成“车队运营总览 → 筛选表格明细”的页面层级。
  - Robotaxi 排队任务显示排队序号和优先级；“可操作任务”改为“当前可触发运维 / 无可触发任务”。
  - 清洁、充电、维修、故障处理、退役任务接入统一分组详情，新增“位置与行驶”分组，展示 Robotaxi 当前单元格、当前位置摘要、当前位置详情、目的位置和运营行驶记录。
  - 运维任务分配目的站点成功后自动创建运营行驶记录；待行驶状态的任务行操作改为查看行驶记录。
  - 运营行驶记录页面恢复路径规划、继续行驶、自动到达、正常到达和异常到达等行操作。

- `src/styles.css`
  - Robotaxi 运营状态面板弱化卡片边框，保持后台运营工作台的紧凑信息表达。

- `src/domain/fieldDictionary.js`、`doc/rules/field-dictionary.md`
  - 补齐任务规划策略、执行、结果中的队列策略、综合状态、规划输入输出、触发对象、决策原因和状态枚举中文。
  - 补齐作业开始时间等运维任务详情字段中文。

- `scripts/verify-field-display-contract.mjs`
  - 扩展任务规划策略、执行、结果样本，覆盖嵌套快照字段中文展示合同。

- `scripts/verify-v040-16-fleet-route-display-closure.mjs`
  - 新增回归合同，覆盖 Robotaxi 面板层级、排队序号、运维任务位置字段、目的站点后自动创建行驶记录、待行驶查看行驶记录、运营行驶记录动作恢复和字段展示合同样本。

## 边界

- 本轮修复的是业务单据本身的闭环，不把清洁、充电、维修等运维任务新增接入模拟运行主扫描。
- 模拟运行仍只通过已有业务服务获得成功、失败或无符合 Robotaxi 等业务结果。
- 本轮恢复运营行驶记录的人工推进和自动到达入口，不做完整真实时间工作流调度重构。
