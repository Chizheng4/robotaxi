# v028.1.1 自动运营模拟正常状态机与时效规则一致性修复

## 1. 迭代类型与边界

- 类型：小迭代 / Bug 修复
- 基线：`v028.1`
- 目标版本：`v028.1.1`
- 指标模型：不涉及，继续保持待执行
- 异常能力：保留现有异常状态、人工处理入口和异常操作，本轮不删除、不改名、不改变行为
- 兼容状态：允许旧数据继续读取，但不得继续进入新的自动正常主链

## 2. 根因

v028.1 的时效规则在 `businessTimingCalculator` 中独立手工维护，没有来源于 WorkflowEngine 的唯一状态边定义。与此同时，WorkflowEngine 使用泛化动作 `TRIP_STEP_EXECUTE` 表达路径规划、行驶、上车和下车等不同业务动作，导致执行规则、状态机、时效配置和中文显示发生分叉。

## 3. 唯一正常状态主链

### 3.1 运营准入任务

```text
待分配 → 待检查 → 检查中 → 已完成
```

### 3.2 运营行驶记录与运营投放任务

```text
RouteExecution：待路径规划 → 行驶中 → 已到达 → 已完成
DeploymentTask：待行驶 → 行驶中 → 已到达 → 已完成
```

RouteExecution 是执行源，DeploymentTask 是父单据投影。两者共享同一个路径规划、行驶和到达事件时间，不允许 WorkflowEngine 对同一流程执行两次动作。

### 3.3 履约行驶记录

```text
待路径规划
→ 前往上车点
→ 待客户上车
→ 客户已上车
→ 前往目的地
→ 已到达目的地
→ 已完成
```

对应动作：

|变更前状态|功能操作|变更后状态|耗时方式|
|---|---|---|---|
|待路径规划|规划行驶路径|前往上车点|固定时长|
|前往上车点|履约行驶推进|待客户上车|单 Cell 时长|
|待客户上车|确认客户上车|客户已上车|固定时长|
|客户已上车|规划行驶路径|前往目的地|固定时长|
|前往目的地|履约行驶推进|已到达目的地|单 Cell 时长|
|已到达目的地|确认客户下车|已完成|固定时长|

### 3.4 服务订单

```text
待估价
→ 待呼叫 Robotaxi
→ 待分配
→ 前往上车点
→ 待客户上车
→ 客户已上车
→ 前往目的地
→ 已到达目的地
→ 结算中
→ 待支付
→ 已完成
```

- 待分配 → 前往上车点：订单匹配成功并创建待路径规划的 Trip。
- 前往上车点至结算中：由 Trip 状态事件同步投影，时间以 Trip 为源。
- 结算中 → 待支付：执行结算。
- 待支付 → 已完成：执行支付。

“待分配 → 前往目的地”与业务顺序冲突，本版本按“待分配 → 前往上车点”实现。

## 4. 唯一工作流状态边注册表

新增一个正式注册表作为以下能力的共同来源：

1. WorkflowEngine 自动动作查询；
2. 工作流时效配置初始化；
3. 业务状态时间线计算；
4. 状态机一致性自动验证；
5. 前端动作与中文名称核对。

注册表区分：

- `DIRECT`：执行引擎直接执行；
- `PROJECTED`：继承源对象同一因果事件，不独立执行；
- `COMPATIBILITY`：只兼容旧数据，不属于新正常主链；
- `EXCEPTION`：异常或人工处理能力，本轮保持不变。

## 5. 时效配置修复

1. 增加 DeploymentTask 三条可见配置，并与 RouteExecution 对应规则使用同一时效规则组。
2. 去除正常配置中的 `VEHICLE_ASSIGNED`、`PENDING`、`TRIP_START` 和 `TRIP_DEPART`。
3. Trip 初始状态统一为 `WAITING_ROUTE`。
4. 增加确认客户上车、确认客户下车时长。
5. 两次路径规划统一使用 `ROUTE_PLAN / 规划行驶路径`。
6. ServiceOrder 的 Trip 驱动状态显示为继承规则，与 Trip 来源规则共享时长，不能配置出不同结果。
7. WTR 编号只作为记录编号，不承担稳定业务语义；状态边业务键作为唯一识别。

## 6. 字段字典收敛

本轮处理与当前流程直接相关的命名：

- `routeExecution` 统一为“运营行驶记录”；
- `WAITING_CUSTOMER_BOARDING` 统一为“待客户上车”；
- `CUSTOMER_ONBOARD` 统一为“客户已上车”；
- `ROUTE_PLAN` 统一为“规划行驶路径”；
- 新增“确认客户上车”“确认客户下车”“继承上游事件”等中文枚举。

同时增加字段上下文枚举能力，修复 `STOPPED`、`BLOCKED`、`MANUAL` 这类相同代码值在不同字段下中文含义互相覆盖的问题。空间对象的通用字段命名重构不在本次 Bug 修复中扩大处理。

## 7. 验证

- 注册表中的正常状态边与 WorkflowEngine 查询结果一致；
- 默认时效配置覆盖全部正常直接边和投影边；
- 同一时效规则组修改后所有关联对象值一致；
- DeploymentTask 与 RouteExecution 同步时间一致；
- ServiceOrder 与 Trip 的接驾、上车、送达、下车时间一致；
- 自动正常流程不再生成 `VEHICLE_ASSIGNED`、`PENDING`、`ASSIGNED`、`PASSENGER_ONBOARD` 或 Trip `SETTLING`；
- 已有异常状态、异常操作和人工处理入口仍存在；
- 字段字典无本轮新增重复冲突；
- bundle、页面、1280 × 720 与 1440 × 900 验证通过。
