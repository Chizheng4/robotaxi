# v041.4.2 手机登录键盘周期修复

## 问题

手机登录页首次打开键盘可以上移，但重复聚焦输入框时会在旧位置上继续累计上移，键盘关闭后也可能保留历史位置。

## 根因

- 手机浏览器已经通过可视视口平移保证输入框可见，登录页又重复使用 `visualViewport.offsetTop` 做顶部补偿。
- `focusout` 早于键盘视口恢复，压缩后的高度可能被错误记录为新的稳定高度。
- 登录容器的历史滚动位置没有在完整键盘周期结束时清理。

## 修复

- 浏览器负责可视视口平移，登录页只根据键盘开关状态调整自身布局。
- 键盘关闭过渡期间保持原稳定高度，视口恢复后再结束键盘状态。
- 键盘关闭与横竖屏切换时统一复位登录容器滚动位置。
- 状态机测试覆盖两次完整键盘周期和周期内多次浏览器平移。

## 边界

本轮只修改统一响应式视口服务和登录页键盘布局，不修改登录身份、业务页面、运行态或模拟运行。
