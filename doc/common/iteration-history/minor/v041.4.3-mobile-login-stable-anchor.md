# v041.4.3 手机登录稳定锚定

## 现场问题

Safari 和微信 WebView 真机输入时仍会把登录内容反复推向顶部，微信顶部工具栏可能遮挡标题，页面出现明显长滚动区域。

## 根因

手机浏览器本身会缩小或平移 visual viewport，并自动滚动聚焦输入框。旧登录页同时修改可视高度、键盘底部空间和表单位置，形成浏览器原生滚动与应用布局重排的双重控制。可视高度已经扣除键盘后，再增加键盘高度 padding 还会人为制造长滚动页面。

## 工程方案

- 登录页采用稳定、固定、不可滚动的首屏画布。
- 手机表单固定在上半部安全区，键盘状态不参与定位计算。
- 浏览器独立负责键盘和输入焦点，不再由页面模拟键盘避让。
- 取消自动聚焦和强制布局视口重排声明。
- 平台统一视口服务继续服务复杂工作台，不承担登录表单定位职责。

## 验证合同

- 登录页不得使用键盘高度生成额外 padding。
- 手机登录表单键盘开关前后使用同一 top 锚点。
- 页面不得自动聚焦，也不得声明 `interactive-widget=resizes-content`。
- 保留手机页面加载、无全局横向溢出和业务工作台回归验证。
