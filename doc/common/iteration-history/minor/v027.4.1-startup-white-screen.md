# v027.4.1 启动白屏修复

## 1. 迭代类型

补丁小迭代。

## 2. 问题与根因

网站启动后出现白屏。4173 端口仍处于监听状态，但启动脚本使用单线程 `HTTPServer`；一个未完成的浏览器连接占用处理线程后，后续 HTML、JavaScript 和 CSS 请求无法得到响应，形成“端口正常、页面不可用”的假启动。

## 3. 旧检查为什么漏检

旧流程只验证 bundle 语法、某一时刻的页面状态和一次 HTTP 200。第一次健康请求可能在阻塞连接出现前通过，已打开页面也无法代表下一次全新启动，因此没有覆盖“连接被占用后关键资源是否仍可加载”。

## 4. 修复内容

1. 静态服务改用 `ThreadingHTTPServer`，每个请求独立处理。
2. 新增启动就绪检查：故意保留一个未完成连接，再读取入口页和五类关键资源。
3. 任一资源超时、状态异常或内容标记缺失时，启动失败并停止新服务。
4. 提交前只静态确认启动器仍使用并发服务，并检查就绪脚本语法。
5. 移除启动时改写 `index.html` 的行为，避免产生无意义的 Git 差异。

## 5. 检查成本原则

不增加浏览器全量回归，也不增加新的长期规则分类。针对性就绪检查实测约 `0.07 秒`，由启动脚本自动执行，不消耗模型上下文；提交前检查只增加脚本语法和并发服务防回退断言。

## 6. 验证结果

- 故意占用一个未完成连接时，入口页和关键资源仍全部返回成功。
- 真实页面完成 React 挂载，业务内容和模拟运行记录正常显示。
- 页面无横向溢出，HTTP 返回 200。
- `zsh -n start-robotaxi.command` 通过。
- `./scripts/check-before-commit.sh` 通过。

## 7. 字段字典影响

本轮不新增或修改业务对象、字段、状态、枚举和中文展示，不需要修改字段字典。
