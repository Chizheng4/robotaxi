# v028.1.2 时效配置权限与计算事件展示修复

## 1. 迭代边界

- 类型：小迭代 / 前端交互与事件可观测性修复
- 基线：`v028.1.1`
- 目标版本：`v028.1.2`
- 不修改工作流状态机、时效计算算法和异常流程

## 2. 继承配置权限

`duration_source_type = INHERITED` 的规则只展示继承结果，不提供编辑入口。用户只能修改 `CONFIGURED` 来源规则；同一 `timing_rule_group_id` 下的继承规则继续自动同步来源值。

验收：继承规则操作列仅保留查看详情，独立配置规则显示调整时长；函数入口同时校验来源类型，避免绕过页面触发编辑。

## 3. 功能命名

- 首次计算：`计算运营模拟时间`
- 已有结果再次计算：`重算运营模拟时间`
- 进行中：`计算中`

该命名直接表达对运营业务节点模拟时间的计算结果，避免向运营用户暴露内部“业务时效”术语。

## 4. 模拟运行事件

计算过程复用现有 `SimulationEvent` 数据结构和模拟运行页事件区，不新增独立日志组件。每次计算生成：

1. `OPERATING_SIMULATION_TIME_CALCULATION_STARTED`：计算开始；
2. `OPERATING_SIMULATION_TIME_CALCULATION_COMPLETED`：计算成功或部分成功；
3. `OPERATING_SIMULATION_TIME_CALCULATION_FAILED`：计算异常。

事件关联当前 SimulationRun，记录计算运行编号、配置编号与版本、对象数、状态变更数、成功数、失败数、真实计算耗时和错误摘要。事件模拟时间沿用运行对象当前模拟时间，真实创建时间使用审计时间。

## 5. 前端展示

- 模拟运行详情删除“时效计算”页签；
- 运行列表继续保留计算状态和进度，便于快速判断结果；
- 计算过程、结果摘要和错误统一在模拟运行事件区展示；
- 人工计算继续使用确认弹窗与完成提示。
