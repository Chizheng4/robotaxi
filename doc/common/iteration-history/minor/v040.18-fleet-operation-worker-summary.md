# v040.18 Fleet Operation Worker Closure And Robotaxi Summary

## 背景

本轮聚焦业务单据本身的底层闭环，而不是模拟运行主路径。目标是修复清洁、充电、维修等运维任务在行驶、到达、作业人员分配和 Robotaxi 摘要展示上的不一致，避免页面层个性化动作破坏服务化闭环。

## 已确认问题

1. 运维任务已自动创建运营行驶记录后，行操作仍可能显示“查看详情”，用户无法直接进入对应行驶记录。
2. 运维任务表格和详情缺少 Robotaxi 当前实时位置、起点位置、目的位置和关联运营行驶记录的完整展示。
3. 运营行驶记录到达后，运维任务仍暴露“已到达目的地”这类中间状态，而不是直接进入待分配作业人员。
4. 待分配作业人员时，清洁 / 维修 / 充电任务可能跳过 Worker 分配直接开始作业。
5. 充电任务缺少“接入充电头”和“断开充电头”两段 Worker 作业闭环。
6. Robotaxi 管理顶部摘要体验不一致，详情展开时空间压缩后信息展示效率低，旧焦点样式出现黑色粗框。
7. Robotaxi 排队任务摘要缺少稳定排队序号，不利于判断任务顺序。

## 设计原则

1. 业务单据生命周期是底层事实，页面只触发服务动作。
2. 运营行驶记录是通用行驶服务，运维任务只能关联和查看统一行驶记录。
3. 运维作业必须先分配作业人员，再进入作业中状态。
4. 充电中的 Robotaxi 不持续占用 Worker；接入完成释放 Worker，充电完成后重新分配 Worker 断开充电头。
5. Robotaxi 摘要只展示关键内容，完整长内容使用悬浮卡片承载，压缩时横向滚动。

## 实现内容

1. `fleetOperationTaskService` 新增 `assignFleetOperationWorker`，统一返回任务、Robotaxi、Worker 三方状态更新。
2. 运维行驶到达或到达确认后，任务状态直接进入待分配作业人员 / 待分配充电作业人员 / 待分配维修作业人员等后续状态。
3. 创建运维运营行驶记录时回写 `origin_cell_id`，前端推导并展示起点位置摘要和详情。
4. 清洁和维修任务改为“待分配作业人员 -> 分配作业人员 -> 待开始作业 -> 作业中 -> 已完成”。
5. 充电任务改为“待分配作业人员 -> 分配作业人员 -> 接入充电中 -> 充电中 -> 待分配作业人员 -> 分配作业人员 -> 断开充电中 -> 已完成”。
6. 前端运维任务关联行驶记录时显示“查看行驶记录”，待分配作业人员时显示“分配作业人员”。
7. 运维任务表格和详情补齐 Robotaxi 当前实时位置、起点位置、目的位置和关联运营行驶记录。
8. Robotaxi 管理顶部重做为两层摘要：车队状态摘要 + 当前 Robotaxi 摘要卡片；排队、位置、可触发动作通过 Popover 展示完整信息。
9. Robotaxi 摘要区域控制高度，支持横向滚动，焦点样式改为轻量 outline，避免黑色粗框。
10. Robotaxi 排队任务摘要和悬浮详情显示排队序号，旧数据无 `queue_sequence` 时按队列位置兜底。

## 模拟运行边界

本轮没有新增模拟运行主路径。模拟运行仍只通过既有业务服务获得业务结果，不在模拟层重写运维任务生命周期。运维任务单自身闭环变更后，模拟运行最多受到“业务动作成功 / 失败 / 无符合 Robotaxi”等结果影响，不新增独立模拟分支。
