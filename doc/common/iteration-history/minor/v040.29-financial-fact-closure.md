# v040.29 业务单据财务事实闭环小迭代

## 背景

本轮处理的问题是：运营准入、运营投放、运营行驶记录、履约行驶记录、服务订单和运维任务在成本 / 收入展示上出现不一致。尤其是清洁任务触发的运营行驶记录成本正常，而运营投放触发的运营行驶记录、准入任务、履约行驶记录、服务订单等对象在详情中显示无或 0。

根因不是单个字段缺失，而是“事实记录生成”和“业务对象摘要回填”没有统一服务化：

- 部分入口只生成 `CostRecord` / `RevenueRecord`；
- 部分入口同时回填业务对象；
- 前端详情读取业务对象摘要字段时，旧对象或未回填对象自然显示为空；
- 运维任务的工作流时效配置只覆盖了 Worker 作业的一部分状态边，未完整覆盖目的站点分配、路径规划、行驶和到达确认。

## 业务边界

业务单据是底层事实来源，模拟运行只是上层统一时间调度。成本、收入和状态时间线必须由业务动作服务沉淀，模拟运行不得再补一套独立闭环。

```mermaid
flowchart LR
  A["人工操作 / 策略触发 / 模拟时间作业"] --> B["业务动作服务"]
  B --> C["业务单据状态变更"]
  C --> D["状态时间线"]
  C --> E["成本 / 收入事实记录"]
  E --> F["业务对象摘要回填"]
  F --> G["前端表格与详情展示"]
  G --> H["经营分析统一数据池消费"]
```

运营行驶记录仍是 Robotaxi 行驶的通用底层服务。运营投放、清洁、充电、维修、退役等任务只应调用同一行驶能力，不应因为触发来源不同而产生成本字段、创建时间或展示行为差异。

## 本轮修复

1. `businessActionService` 的 `withFinancialFacts` 在生成增量成本记录后，将成本摘要同步回填到来源业务对象。
2. 服务订单收入事实生成后同步回填应收、实收、未收收入摘要。
3. `revenueCalculator.createRevenueCalculation` 批量收入计算返回带收入摘要的服务订单投影，供模拟运行结束后的统一收入生成使用。
4. `main.jsx` 展示层从 `cost_records` 和 `revenue_records` 兜底聚合摘要，保证历史快照也能正确显示。
5. `workflowTransitionRegistry` 补齐清洁、充电、维修、退役的目的站点分配、路径规划、行驶推进、到达确认状态边。
6. `fieldDictionary.js` 与 `field-dictionary.md` 补齐新增状态边、成本明细、收入明细和收入生成时间中文展示。
7. 新增专项验证 `scripts/verify-v040-29-financial-fact-closure.mjs` 并接入提交前检查。

## 版本结论

`v040.29` 将成本 / 收入事实记录与业务对象摘要回填收敛到统一合同，修复了同一业务服务因不同任务入口产生展示差异的问题，并保持模拟运行作为上层调度，不改变其主路径和性能边界。
