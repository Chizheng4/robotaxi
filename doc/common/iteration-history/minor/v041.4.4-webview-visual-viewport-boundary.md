# v041.4.4 WebView 可视视口边界

## 现场问题

普通手机浏览器已稳定，但微信 WebView 聚焦输入框时仍可能上移并遮挡标题、发生视觉放大与水平偏移；登录控件宽度贴近屏幕两侧，进入工作台后根页面也可能左右移动。

## 根因

微信 WebView 聚焦时可能同时改变 Visual Viewport 的尺寸、纵向偏移和横向偏移。只固定在 Layout Viewport 的画布不能代表用户当前真实看到的区域。登录面板使用百分比宽度和居中变换，也会在可视窗口缩放后放大边界误差。根页面与业务横向滚动容器缺少明确的所有权区分。

## 工程方案

- 统一视口服务输出 Visual Viewport 的宽、高、左偏移和上偏移。
- 登录画布作为 device-fixed 层投影到真实可见窗口。
- 登录面板使用明确安全边距和最大宽度，不依赖居中 transform。
- 根页面禁止水平滚动，业务表格等指定容器独立拥有横向滚动。
- 禁用 WebKit 自动文字膨胀，不禁止用户主动缩放。
- README 同步当前项目说明、在线地址和一键发布方式。

## 边界

不对微信 UA 写特判，不禁用用户缩放，不修改业务页面字段、业务对象、运行态或模拟运行。
