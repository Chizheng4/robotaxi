# v041.2.14 GitHub Pages 公开演示发布

## 目标

把 Robotaxi 运营平台建设为可通过 GitHub Pages 免费公开访问、可随 `main` 提交持续更新的演示站点，同时保持本地开发、业务闭环和模拟运行不变。

## 边界

1. GitHub Pages 只托管静态应用，不增加后端、账号和共享数据库。
2. 多位访客各自在浏览器中独立运行和保存模拟数据，互不影响。
3. 使用 GitHub 默认域名时不承诺中国大陆网络质量；大陆 CDN 和域名备案属于后续独立版本。
4. 保留 `start-robotaxi.command`，云发布配置不进入业务服务和模拟运行主路径。

- [x] 任务 1：新增最小生产构建，只发布 `index.html`、运行时 `src` 和 `vendor`，排除设计文档、脚本和本地日志。
- [x] 任务 2：使用提交版本刷新入口资源与动态模块缓存，避免网站更新后仍加载旧代码。

## 工程实现

- `scripts/build-github-pages.mjs` 生成独立 `dist` 站点并检查 bundle 与源码一致。
- `.github/workflows/deploy-pages.yml` 使用 GitHub 官方 Pages Actions 自动部署。
- `scripts/verify-github-pages-build.mjs` 检查相对路径、动态模块、缓存版本、发布范围和站点体积。
- `scripts/serve-github-pages-preview.mjs` 在 `/Robotaxi/` 子路径预览生产产物。
- `scripts/verify-browser-load.mjs` 支持设备视口覆盖和手机首屏合同检查。
- `src/main.jsx` 与 `src/styles.css` 增加窄屏工作台行为，不改变业务服务。

## 验证结果

- GitHub Pages 发布产物：6.01 MiB。
- 子路径：`/Robotaxi/` 下入口和全部动态模块可加载。
- 真实浏览器：`1280×720`、`1440×900`、`375×812` 均无白屏和运行时异常。
- 手机视口：导航默认收起，页面无全局横向溢出。
- `bash scripts/check-before-commit.sh` 全量通过。
- 当前仓库尚未配置 GitHub remote，实际公网地址需在 GitHub 建立远程仓库并启用 Pages 后生成。
