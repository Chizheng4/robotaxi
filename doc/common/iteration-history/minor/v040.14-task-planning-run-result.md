# v040.14 任务规划执行结果与策略页展示规范

## 背景

用户指出：任务规划策略也应符合统一的策略对象架构，即“策略配置 / 策略执行 / 策略结果”，并且策略配置表单下方应展示最近策略执行，和运维策略配置等页面保持一致。

本轮同时发现运维调度策略页面没有接入最近执行区，根因是前端 `RecordTable` 只把 `fleetOperationPolicies` 和旧 `taskDispatchStrategies` 识别为策略配置页，判断条件写窄，导致同类策略对象展示不一致。

## 本轮方案

1. 任务规划策略补齐执行与结果对象。
2. 页面刷新、按钮可用性预览不生成执行记录。
3. 真实业务触发生成任务规划执行和结果。
4. 运维策略配置、运维调度策略、任务规划策略统一展示最近策略执行。

## 变更

- `src/domain/robotaxiTaskPlanningTypes.js`
  - 新增 `TaskPlanningRunStatus`、`TaskPlanningResultStatus`。
  - 新增 `createRobotaxiTaskPlanningRun`、`createRobotaxiTaskPlanningResult`。

- `src/services/robotaxiTaskPlanningService.js`
  - 新增 `executeRobotaxiTaskPlanning`，封装策略裁决和审计对象生成。

- `src/services/fleetOperationTaskService.js`
  - 运维任务真实触发支持生成任务规划执行/结果。

- `src/services/fleetOperationPolicyService.js`
  - 运维策略扫描聚合任务规划执行/结果。

- `src/data/orderMatchingEngine.js`、`src/services/serviceOrderService.js`
  - 订单匹配候选生成任务规划执行/结果。

- `src/services/businessActionService.js`
  - 自动投放与自动订单匹配通过业务服务沉淀任务规划执行/结果，不让模拟运行单独实现规划逻辑。

- `src/main.jsx`
  - 新增 `robotaxiTaskPlanningRuns`、`robotaxiTaskPlanningResults` 运行态集合、页面、保存恢复和重置。
  - 任务规划策略页面下方展示最近任务规划执行。
  - 运维调度策略页面下方展示最近运维调度执行。

- `src/domain/fieldDictionary.js`、`doc/rules/field-dictionary.md`
  - 同步任务规划执行/结果字段和枚举中文。

## 边界

- 本轮不将任务规划执行/结果加入模拟运行主扫描。
- 任务规划执行/结果是业务审计事实，不驱动状态推进。
- 页面预览不落库，只有真实业务触发落库。
