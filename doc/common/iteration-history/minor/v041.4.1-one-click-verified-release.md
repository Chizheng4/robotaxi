# v041.4.1 一键验证发布流程

## 目标

把版本提交、Contribution、版本标签、GitHub Actions、GitHub Pages 和公网最新版验证收敛为一条稳定流程，用户在 Codex 完成版本提交后只需双击一次发布文件。

## 已完成

- `main` 的正式版本提交稳定触发一次 Pages 工作流，Actions 标题使用版本提交说明。
- 工作流校验 `VERSION.md`、HEAD 标签和提交标题一致。
- 一键发布先运行完整检查，再推送 `main` 与标签。
- 推送后等待对应提交的 Actions 成功，并校验公网部署清单已经切换到当前版本与提交。
- 删除旧的重复版本创建入口，只保留 `publish-robotaxi.command`。
- 发布文档明确 Codex 提交、用户发布和 Contribution 的边界。

## 边界

本版本只调整工程发布流程，不修改前端业务功能、业务对象、状态机、运行态或模拟运行。
