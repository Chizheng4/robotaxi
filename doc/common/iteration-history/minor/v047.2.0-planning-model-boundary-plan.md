# v047.2.0 经营规划模型边界迭代计划

## 目标

统一长期需求预测、经营目标比较、计划承接、供应决策和生产质检时间轴，确保字段、公式、快照、页面和计划单据使用同一业务语义。

## 任务

1. 预测周期从执行快照读取周期单位、数量和开始日期，按自然日历计算包含式结束日期。
2. 预测结果保留市场预测、经营目标、规划模式和计划承接量，完整展示比较公式及快照来源。
3. 生产趋势拆分生产完成与质检合格可供给两个时间轴；剩余缺口只由质检合格量冲减。
4. 服务能力同时展示计划需求、有效容量和缺口；零缺口表达计算结果，不伪造瓶颈。
5. 供应决策保存覆盖缺口、安全容量和最终供给数量，生产计划可追溯供应决策执行。
6. 计算过程改用标准查看按钮，保持桌面和手机端一致。
7. 同步字段字典、模型验证、前端合同、Bundle 和真实页面验收。

## 验收

- 12 个月自然周期不再按固定 360 天计算。
- 预测结果能解释规划模式如何形成计划承接量。
- 生产完成和质检合格趋势不再同日起点或互相覆盖。
- 缺口 43、安全产能 5% 时明确得到 43 + 3 = 46。
- 计算过程入口、图表和数据分解在桌面及手机端可用。

## 完成结果

- 领域模型、供应决策服务、字段字典、预测结果页面和共享图表控件已统一。
- 新增 v047.2.0 模型边界验证并纳入提交前检查。
- 提交前全量检查、桌面真实页面和 390×844 手机真实页面均通过。
