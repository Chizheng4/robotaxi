# v040.15 经营分析指标池与 Robotaxi 管理体验修复

## 背景

用户反馈四类问题：

1. 经营分析中财务结果、服务效率、过程质量在不同页面显示不一致。
2. Robotaxi 管理页仍未体现已确认的综合状态、位置、占用和排队任务体验优化。
3. Robotaxi 触发运维任务后页面响应偏慢。
4. 运维任务状态展示仍像旧版本，没有体现新闭环语义。

## 根因

- 经营分析洞察栏把当前页面过滤后的指标行当成全量指标池使用，导致页面主题之外的指标被误隐藏。
- Robotaxi 列表仍主要依赖通用表格行操作，缺少面向真实运营的状态总览；行操作渲染时也重复调用任务规划预览。
- 运维任务底层状态机已有“排队、分配目的站点、待行驶、前往目的地、待分配作业人员、作业中、完成”的服务化闭环，但字段字典和状态展示文案仍保留旧语义。

## 变更

- `src/main.jsx`
  - 经营分析洞察栏改为读取同一统计周期下的统一指标池。
  - Robotaxi 管理页新增运营状态面板，展示可运营、不可运营、执行中、有排队任务、当前位置、当前占用、排队任务和可操作任务。
  - Robotaxi 行操作复用缓存后的任务规划预览结果，减少重复计算。
  - 运维任务状态按任务类型细分显示“待分配清洁站 / 充电站 / 维修站”等闭环文案。

- `src/styles.css`
  - 增加 Robotaxi 运营状态面板样式，保持现有后台工作台密度和表格详情结构。

- `src/domain/fieldDictionary.js`、`doc/rules/field-dictionary.md`
  - 同步更新任务排队、目的站点、作业人员分配和维修中等状态中文。

- `scripts/verify-v040-15-metric-robotaxi-fleet-ui.mjs`
  - 新增回归合同，覆盖经营分析统一指标池、Robotaxi 运营面板、任务规划预览缓存和运维状态展示。

## 边界

- 本轮不把清洁、充电、维修等运维任务接入模拟运行主扫描。
- 模拟运行仍只调用已有业务服务并消费业务结果。
- 本轮不改变运维任务底层状态枚举，只修复展示语义和前端体验。
