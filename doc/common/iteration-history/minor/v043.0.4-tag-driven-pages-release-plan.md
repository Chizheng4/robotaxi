# v043.0.4 GitHub Pages 标签发布迭代计划

状态：已完成

## 目标

消除提交说明、标签与 main 分步推送造成的发布歧义，并让终端网络不可用时快速停止。

## 任务

- [x] 将 Pages workflow 改为版本标签触发；
- [x] 使用标签版本号生成 Actions 标题；
- [x] 使用 VERSION、Tag 和 Tag/HEAD 关系校验发布事实；
- [x] 调整 publish 为先 main 后 Tag；
- [x] 增加 GitHub HTTPS 快速网络预检；
- [x] 更新发布合同、版本记录和站内更新记录。

## 验收

- Commit Message 不再影响发布结果；
- Actions 标题稳定显示发布标签；
- 只有版本 Tag 能自动触发 Pages 部署；
- Terminal 无法连接 GitHub 时在 15 秒内停止；
- 发布合同和完整提交前检查通过。

## 验证结果

- 本地版本、Tag/HEAD 和 workflow 静态合同通过；
- 网络预检失败分支可输出明确原因；
- 业务代码、页面功能和模拟运行均未修改。
