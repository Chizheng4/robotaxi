# v036.4 经营指标统一刷新体验

## 版本定位

本次作为 `v36.x` 的小迭代，修正经营分析页面的数据刷新心智。

核心目标：经营指标总览、财务表现分析、服务经营分析和运营过程诊断共用同一个经营指标结果池。前端菜单只负责按角色和分析层级展示，不给用户造成“每个页面都要单独计算一次”的理解负担。

## 当前问题

1. 四个经营分析页都显示“计算经营周期指标”，用户容易误解为页面级计算。
2. 指标页面虽然共用 `metricObservations`，但前端文案没有明确“统一经营数据池”。
3. 指标结果筛选没有优先按当前统计周期选择，用户切换周期时容易不确定当前看到的是哪一批结果。
4. 空状态仍提示“先计算经营周期指标”，不符合经营分析页作为展示层的定位。

## 目标设计

- 统一入口文案改为“更新经营数据”，表达一次刷新统一指标数据池。
- 四个经营分析页共用同一批 `MetricObservation`，只按指标分组展示不同视角。
- 当前统计周期决定分析页优先展示的指标结果；若当前周期没有结果，则显示空状态。
- 指标体验面板轻量展示最近更新批次、更新状态、统计周期、来源记录和质量提示。
- 指标计算记录、指标观测仍归属“数据计算管理”，作为追溯和排查页面。

## 验收标准

- 经营分析页不再出现“计算经营周期指标”作为主操作文案。
- 主操作统一为“更新经营数据”，四个分析页都刷新同一指标结果池。
- 当前统计周期优先控制经营分析页展示结果。
- 面板显示最近更新批次和状态，帮助用户知道当前数据是否已刷新。
- 提交前检查通过，页面可正常加载。
