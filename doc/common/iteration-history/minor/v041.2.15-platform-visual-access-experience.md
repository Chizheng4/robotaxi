# v041.2.15 平台视觉与演示访问体验

## 目标

在不改变业务页面结构、业务服务、模拟运行和本地数据的前提下，升级公开演示站点的视觉框架、轻量登录体验和版本记录入口。

## 边界

1. 登录仅作为公开演示的前端访问门，不实现账号体系，不写入业务运行态。
2. 登录角色固定为“金星”，退出、十二小时会话到期或会话不可识别时返回登录页。
3. 视觉升级复用全局设计令牌和既有页面结构，不为单页建立私有视觉体系。
4. 迭代记录由 `VERSION.md` 生成，不在页面组件中重复维护版本内容。

## 完成任务

- [x] 任务 1：新增独立访问会话服务和全页登录入口，支持按钮与回车提交、错误提示、会话到期和退出。
- [x] 任务 2：新增版本记录生成脚本和运行时数据模块，并接入提交前一致性检查。

- [x] 任务 5：保持页面信息架构和业务功能不变，登录与版本展示不接入模拟运行和业务运行态。
- [x] 任务 6：重建 bundle，完成语法、工程合同、GitHub Pages 构建和桌面/手机真实浏览器验证。

## 工程实现

- `src/ui/platformExperience.js` 负责轻量访问会话的创建、校验、过期与清理。
- `scripts/generate-release-history.mjs` 从 `VERSION.md` 生成 `src/ui/releaseHistory.js`，页面只消费生成数据。
- `PlatformEntry` 将访问体验与业务 `App` 分离；退出只清理访问会话，不清理业务数据。
- `src/styles.css` 使用全局灰蓝设计令牌协调品牌区、顶部、导航和登录画布，关键数据区继续使用白色。
- `scripts/verify-browser-load.mjs` 自动通过登录入口后验证工作台，保持原有桌面与手机合同。

## 验证结果

- 迭代记录浮层可打开、关闭和滚动，最新版本与 `VERSION.md` 一致。
- `1440 x 900` 视觉对比通过；`1280 x 720` 与 `375 x 812` 页面加载和响应式合同通过。

- `bash scripts/check-before-commit.sh`、GitHub Pages 构建验证和真实浏览器加载验证通过。
