# v045.2.0 页面架构合同与详情控件强约束

## 目标

- 让所有页面通过代码注册声明展示框架、资源性质、详情、操作和事件区。
- 让新增页面无法绕开统一表格、详情、操作隔离和响应式控件。
- 让没有专属详情设计的对象自动获得一致、可读的语义分组详情。

## 完成任务

- [x] 审计全部 78 个前端叶子页面及其控件接入情况。
- [x] 建立页面架构注册表并覆盖全部导航页面。
- [x] 增加导航与页面架构一一对应的启动校验。
- [x] 增加页面操作声明与实际操作列一致性校验。
- [x] 将通用详情升级为基本信息、业务信息、关联与结构语义页签。
- [x] 收敛策略、单据、结果和运行页面的事件区语义。
- [x] 增加提交前自动合同检查和桌面、手机真实浏览器回归。

## 边界

- 未修改业务对象、状态机、字段字典、业务服务或模拟运行闭环。
- 页面架构注册表只管理前端展示合同，不承载业务判断。
