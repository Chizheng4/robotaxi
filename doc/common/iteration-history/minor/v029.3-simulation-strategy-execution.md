# v029.3 小迭代：自动模拟真实业务策略执行

## 背景

自动运营模拟需要模拟完整业务工作流，而不是只把业务单据推进到下一个状态。当前问题是部分动作只写状态，不复用人工点击时的业务策略：价格预估没有生成真实 Route，履约和投放路径没有沉淀路径规划事实，后续经营成本、收入、指标与预测会缺少可信的时间、位置、距离和路径基础。

## 目标

1. 自动模拟执行引擎中的关键动作必须复用真实业务服务和策略结果。
2. 价格预估、履约行驶、运营投放都必须生成 Route 和 RoutePlanningRun。
3. 履约行驶记录的初始位置必须来自匹配到的 Robotaxi 当前格子。
4. 投放行驶必须按路径步骤推进 Robotaxi 位置、行驶距离、剩余距离和到达状态。
5. 结算必须能消费履约行驶沉淀的实际距离和时长。
6. 新增低成本自动验证，防止再次回退为“只改状态”的模拟。

## 执行内容

- 新增 `src/services/routePlanningService.js`，集中提供价格预估路径、运营投放路径、履约路径、路径执行步进和到达确认能力。
- 更新 `src/services/simulationHandlers.js`：
  - 价格预估动作生成价格预估路径和路径规划运行记录。
  - 订单匹配后创建履约行驶记录时使用 Robotaxi 当前格子。
  - 履约路径规划生成接驾路径和送达路径，写入 route_history。
  - 投放任务创建写入目标服务区、目标格子、投放任务和运营行驶记录。
  - 投放路径规划、行驶步进、到达确认复用统一路径服务。
  - 结算动作复用最终费用计算服务，并写入最终定价运行和决策。
- 更新 `src/main.jsx` 执行引擎上下文，补齐 routes 和 routePlanningRuns 的状态读写。
- 修复 `src/domain/serviceOrderSettlement.js` 的履约耗时解析，支持 `时:分` 和 `时:分:秒`，并在订单实际时长为 0 时回退履约记录。
- 新增 `scripts/verify-simulation-strategy-execution.mjs`，用小型封闭地图验证价格、匹配、履约、结算、投放和行驶步进。
- 更新提交前检查，把模拟处理器、路径服务和新增业务验证纳入自动检查。

## 验证

- `node --check src/services/simulationHandlers.js`
- `node --check src/services/routePlanningService.js`
- `node --check scripts/verify-simulation-strategy-execution.mjs`
- `node scripts/verify-simulation-strategy-execution.mjs`

完整提交前检查和前端 bundle/page 验证在提交前执行。

## 风险与后续

- 本次仍以正常主链为目标，不扩展异常流自动模拟；异常状态和人工异常能力不被移除。
- 当前服务层仍在前端域内，后续如果迁移后端，应继续保持 ExecutionEngine 调用统一业务服务，而不是复制 UI 逻辑。
- 成本、收入和经营指标后续应优先消费 Route、RoutePlanningRun、Trip、RouteExecution 等事实数据，避免再从状态名反推业务过程。
