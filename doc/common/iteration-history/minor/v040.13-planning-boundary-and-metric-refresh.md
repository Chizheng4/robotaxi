# v040.13 任务规划边界与经营数据自动刷新

## 背景

用户确认：任务规划策略应覆盖 Robotaxi 当前无队列、有队列、正在执行任务和空闲等所有任务进入与排序场景。空闲不是另一个策略域，而是当前任务队列长度为 0 的综合状态输入。

同时，模拟运行不能实现新的业务闭环，只能在业务事实完成后调用已有业务服务。经营分析管理应消费同一经营数据池，不能让不同分析页面表现为各自独立计算。

## 本轮方案

1. 任务规划策略作为 Robotaxi 任务准入、排队和优先级的唯一策略入口。
2. 旧任务调度策略页面隐藏保留兼容，不再作为前台核心入口。
3. 任务优先级配置由任务规划策略派生，避免任务调度策略与任务规划策略重复决定“Robotaxi 下一步做什么”。
4. 模拟运行完成后继续自动触发成本和收入事实生成；成本和收入进入终态后，再调用统一指标计算入口刷新经营数据池。
5. 重置模拟数据时同步清空自动财务和自动经营数据刷新锁。

## 边界

- 本轮不把清洁、充电、维修等运维任务加入模拟运行主扫描路径。
- 模拟运行不直接生成经营指标，只触发既有统一经营数据服务。
- 旧任务调度对象和历史快照字段不删除，避免旧数据加载失败。
