# v025.8 模拟事件展示与通用布局修复

## 迭代目标

基于 v025.7 已打通的模拟闭环，收敛前端重复展示和通用交互问题。

本轮只处理模拟运行管理、事件展示、表格事件区布局和详情操作，不扩展新的业务闭环。

## 用户输入

1. 模拟运行管理下方最近事件与模拟事件记录页面重复，评估并去除重复模块。
2. 下方最近事件区域拖大时会覆盖上方表格和分页，需要形成通用布局框架。
3. 模拟运行表格对象的查看详情和右侧详情栏应复用通用能力。
4. 模拟运行页黑底诊断区多余，应去除。
5. 增加清空历史事件能力，只清空过程事件，保留业务层数据。

## 实现内容

1. 去除“模拟事件记录”独立菜单入口和工作台恢复入口。
2. 保留 `SimulationEvent` 数据模型和模拟运行页下方事件面板。
3. 新增“清空模拟事件”按钮，仅清空 `simulationEvents`。
4. 移除模拟运行页黑底调试诊断区。
5. 表格区新增统一 `record-table-section` 容器，并基于实际容器高度计算表格滚动高度。
6. 操作列新增统一选中捕获层，点击行操作时也会选中当前对象，右侧详情保持通用上下文。

## 验证要求

1. 重新编译 `src/main.bundle.js`。
2. 检查 bundle 语法。
3. 运行提交前检查。
4. 浏览器验证：
   - 模拟事件记录入口不再显示；
   - 模拟运行页仍可查看模拟事件；
   - 清空模拟事件不影响业务数据；
   - 事件区拖动不覆盖表格分页；
   - 点击模拟运行记录或查看详情后右侧详情可显示。
