# v049.5.0 城市空间层级与底图参考

## 目标

让广州城市地图中的一级 Zone、可选二级 Zone、Place 和 ServiceArea 按真实几何形成可验证的包含关系，并继承当前地图视觉建立按缩放层级逐步呈现的规划体验。

## 任务

- 建立多边形闭合、自交、父子包含和完整候选目录校验。
- 绘制子对象前聚焦直接父级，降低跨界绘制和层级误选。
- 从当前 MapLibre 底图识别道路、建筑、地点、水域等紧凑参考，保存到运营区域方案快照。
- 纯水域服务区域阻止发布；缺少道路参考时给出可处理提示。
- 按城市、一级 Zone、二级 Zone、Place、ServiceArea 和道路视野建立语义缩放。
- 保持手机端只读、网格仿真业务运行和当前 OpenFreeMap 视觉不变。

## 验收

- 二级 Zone、Place 或 ServiceArea 越过直接父级边界时无法发布。
- 自交多边形和纯水域服务区域无法通过校验。
- 方案可以追溯绘制时识别的底图参考，但不会把底图要素冒充业务对象。
- 缩放地图时只显示当前层级需要的对象和标签。
- 修改 `src/main.jsx` 后 Bundle 一致，完整提交前检查和真实浏览器加载通过。

## 结果

- 空间拓扑服务验证父子包含、自交边界、纯水域服务区域和完整候选目录，专项合同通过。
- 地图规划会保存范围内底图要素的紧凑快照，业务填充位于地图原生文字下方。
- 完整提交前检查通过，GitHub Pages 产物为 10.55 MiB。
- Codex 浏览器完成桌面规划面板与手机只读地图验收；无横向溢出，手机端不挂载规划入口。
- 无界面真实浏览器加载检查通过，未出现白屏或运行异常。
