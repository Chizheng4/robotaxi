# v028.1.3 状态时间线创建动作中文化修复

## 1. 问题与根因

运营准入任务、运营投放任务、运营行驶记录、服务订单和履约行驶记录共用 `action_type` 字段。字段不存在重复定义，问题是时间线创建记录使用 `objectType.toUpperCase()` 动态拼接：前四类生成了 `READINESSTASK_CREATE`、`SERVICEORDER_CREATE` 等非标准枚举；`TRIP_CREATE` 虽为标准值，但统一值字典缺少中文。

## 2. 修复方案

新计算结果使用固定标准动作：

|业务对象|标准动作|中文显示|
|---|---|---|
|运营准入任务|READINESS_TASK_CREATE|创建运营准入任务|
|运营投放任务|DEPLOYMENT_TASK_CREATE|创建运营投放任务|
|运营行驶记录|ROUTE_EXECUTION_CREATE|创建运营行驶记录|
|服务订单|SERVICE_ORDER_CREATE|创建服务订单|
|履约行驶记录|TRIP_CREATE|创建履约行驶记录|

已有历史计算结果继续保留四个旧拼接值，但在 `action_type` 字段上下文中兼容翻译为相同中文，不要求用户重新计算。

## 3. 验证与完成

- 验证五类时间线首条记录均写入标准动作；
- 验证五个标准值和四个历史兼容值均显示中文；
- 同步文档与代码字段字典；
- 重新构建 bundle，执行提交前检查；
- 归档方案，提交并创建 `v028.1.3` 标签。
