# v049.7.0 城市空间规划闭环修正

## 目标

修复城市空间规划“看得到但难以真正使用”的根因：规划视角不得替用户强制选择对象，旧版固定范围不得继续冒充有效对象，边界编辑必须具备可操作顶点，新建、编辑和停用必须各自表达清楚。

## 业务与产品边界

- 广州受控演示范围是地图数据集参考，不是业务 Zone，不参与画像、预测、供应、单据或模拟运行。
- 地图缩放只推荐空间对象；用户按“空间对象 → 类型 → 名称”最终确认。
- Zone、Place 和 ServiceArea 的父子关系由完整几何校验决定，页面选择不能绕过空间事实。
- 新建从空草稿开始；编辑从已发布对象进入；停用属于已发布对象管理，三者不能混为一个操作。
- 旧版空间方案保留历史，但只有当前合同版本可以进入有效城市目录。

## 验收标准

- 低缩放能识别广州受控演示范围，但不会把它选成业务对象。
- 打开规划时系统给出推荐，用户可以自行选择运营区域、地点或服务区域。
- 一级 Zone 与二级 Zone 的非法包含关系不能发布。
- 新建不会自动载入已选对象；编辑所选对象后可拖动边界顶点并形成新版本草稿。
- 旧版固定区域不再出现在当前城市目录，历史记录仍保留。
- 手机不出现复杂规划入口；网格仿真对象、单据和模拟运行保持不变。

## 验证结果

- 自动验证覆盖当前合同隔离、广州范围参考、三类空间对象、两级 Zone、几何归属、停用保护与双地图场景隔离。
- 桌面真实操作已创建并发布运营区域；地图标签可选中对象，编辑模式可显示并拖动顶点和中点，调整后的边界可形成新草稿并重新识别底图要素。
