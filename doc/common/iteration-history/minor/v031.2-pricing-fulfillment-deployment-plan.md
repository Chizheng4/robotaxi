# v031.2 定价履约口径与投放目标模型小迭代方案

状态：已完成。

版本定位：小迭代。

核心目标：让定价、服务订单和履约事实使用同一套清晰字段；同时把临时运营投放目标从固定点改为受范围约束的随机模型，避免“当前位置投放到当前位置”。

---

## 1. 字段与对象职责

1. `estimated_price` 是预估价格的主字段，前端不再展示“客户报价”作为主概念。
2. `quoted_price` 保留为兼容字段，不作为新增页面展示和计算主口径。
3. 服务订单不再展示 `actual_distance_km / actual_duration_min`，改为展示：
   - `trip_total_distance_km`：履约总距离（公里）；
   - `trip_total_duration_min`：履约总时长（分钟）；
   - `final_price`：最终价格。
4. 定价结果新增：
   - `fulfillment_distance_km`：履约距离（公里）；
   - `fulfillment_duration_min`：履约时长（分钟）；
   - `final_price`：最终价格。
5. 履约行驶记录 `time_elapsed` 前端显示为“已耗时（分钟）”，表达整条 Trip 的累计耗时。

---

## 2. 定价与结算逻辑

1. 预估定价阶段以路径结果为输入：
   - `estimated_distance_km = Route.total_distance_km`；
   - `estimated_duration_min = ceil(Route.route_step_count * 6 秒 / 60)`。
2. 6 秒 / Cell 来自 50m Cell 与 30km/h 车速近似，当前作为最小模型常量；后续可迁移到模拟配置。
3. 最终结算阶段只使用履约事实：
   - `fulfillment_distance_km = Trip.total_distance_km`；
   - `fulfillment_duration_min = Trip.time_elapsed`；
   - 价格仍沿用预估时保存的起步价、距离单价、时间单价和动态系数快照。

---

## 3. 模拟运行状态展示

1. 保留 `DRAINING` 状态，因为它代表计划 Tick 已结束、但已触发工作流仍在自动执行。
2. 前端中文由“排空中”调整为“收尾执行中”，避免误解为运行已经完全结束。
3. `FAILED` 仅表示 SimulationRun 自身无法完成或排空超限；普通动作失败继续记录在 SimulationEvent。

---

## 4. 投放目标临时模型

在需求预测和供给计划模块完成前，投放目标使用受约束随机模型：

1. 目标必须来自已定义的 ServiceArea 可用车辆点位；
2. 目标不能等于当前 Robotaxi 所在 Cell；
3. 优先选择与当前位置存在距离的目标；
4. 使用 Robotaxi 编号和当前候选集生成稳定随机选择，避免固定投向单一服务区。

---

## 5. 执行结果

1. 预估定价改为从路径移动步数按 6 秒 / Cell 换算预估时长，并保存路径步数和时长口径快照。
2. 服务订单展示与结算主路径改为 `estimated_price / final_price / trip_total_distance_km / trip_total_duration_min`。
3. 最终定价决策保存 `fulfillment_distance_km / fulfillment_duration_min / final_price`，不再依赖历史 actual 字段作为主口径。
4. Trip 与 RouteExecution 行驶耗时统一按 Cell 步进累加，字段中文统一为“已耗时（分钟）”。
5. 运营投放目标改为按车辆当前位置从服务区车辆可用点位中受约束选择，避免投放到当前 Cell。
6. 收入、成本和定价校验中的金额/距离优先级调整为新字段优先，历史字段仅兼容兜底。
