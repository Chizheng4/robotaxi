# v049.8.0 城市空间规划可靠性

## 目标

从根源修复区域边界不真实、绘制偶发无效和底图要素采集不稳定，使“绘制 → 完成 → 采集 → 校验 → 发布”成为可重复验证的城市空间规划闭环。

## 任务

- [x] 用广州市行政边界多边形快照替代地图包围盒矩形，并保持其与业务 Zone 隔离。
- [x] 绘制完成改为确定性适配器调用，不再伪造键盘事件。
- [x] 地图或瓦片未就绪、点位不足、参考要素为空时阻止形成无效草稿并返回中文原因。
- [x] 底图参考采集同时读取矢量数据源与可见图层，并保存紧凑来源快照。
- [x] 增加行政边界、绘制闭合、空结果保护和重复规划服务验证。
- [x] 完成桌面真实绘制、校验、发布的重复操作验收，验证广州边界显示、网格回归和手机只读。

实际验收记录：桌面端连续验证越界与冲突拦截、合法区域底图要素采集、校验、发布及刷新恢复；服务验证重复执行 25 轮；网格仿真可正常切换，手机端城市地图保持只读且不提供规划入口。

## 验收标准

- 低缩放能清晰识别广州市行政范围，边界不是矩形且不会被当作业务 Zone。
- 连续执行绘制、完成、校验和发布不会出现偶发无响应或重复回调。
- 地图数据未就绪时页面说明原因，不生成没有底图参考的方案。
- 已发布对象刷新后仍存在并可选中；网格仿真业务和手机只读能力不受影响。
