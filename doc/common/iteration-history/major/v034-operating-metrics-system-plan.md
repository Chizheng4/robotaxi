# v034 经营指标系统大版本计划

## 状态

v034.8 已完成。当前已将指标计算从单次模拟运行口径修正为经营统计周期口径，并明确经营分析页面采用分析型页面骨架，不再沿用业务对象列表页筛选结构。

## 目标

基于现有模拟业务闭环、成本事实和收入事实，建立经营指标系统，使平台能够从模拟运行结果中生成可解释、可下钻、可复盘的经营指标。

## 前端体验要求

1. 不做营销式大屏，不使用大标题、厚重色块或装饰背景。
2. 指标总览必须优先服务经营者、CEO 和投资人视角，先看经营结果，再看链路解释，最后查看数据可信度和明细。
3. 指标卡片用于表达单个核心指标，避免卡片套卡片；计算记录和观测明细默认弱化或收起，不占据主阅读区域。
4. 趋势、构成、异常、质量提示和详情必须能互相解释。
5. 所有指标名称、字段名、状态和枚举必须来自统一字段字典。
6. 指标详情必须展示：定义、公式、来源字段、窗口、质量状态、来源对象。
7. 经营分析页不显示状态分类、关键词查询和状态筛选；这些控件属于业务对象、单据和记录管理页。

## 执行原则

1. 先确认方案，再编码。
2. 每个子版本独立提交和打标签。
3. 任何新增指标对象、字段、枚举必须同步字段字典文档和前端字段字典。
4. 前端必须复用经营分析管理的统一工作台骨架。
5. 指标计算只消费事实，不修改业务单据生命周期；经营指标默认按经营统计周期聚合，不隶属于单个模拟运行对象。
6. 不进入预测模型、潜在需求、车辆状态历史快照等 P1/P2 能力，除非用户明确确认。

## 暂停条件

进入编码后如出现以下情况必须暂停确认：

1. 指标字段命名存在多种合理选择。
2. 成本或收入事实与业务单据对账不一致。
3. 指标页面需要新增超出经营分析管理的导航结构。
4. 自动计算触发会造成性能风险。
5. 需要新增车辆状态历史快照、潜在需求事实等 P1 能力。
