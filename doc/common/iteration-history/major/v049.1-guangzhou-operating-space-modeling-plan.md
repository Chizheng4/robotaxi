# v049.1.0 广州运营空间建模迭代计划

## 目标

在保留现有模拟网格地图与全部业务逻辑的前提下，使地理地图成为可直接触发的运营空间建模工具，跑通“绘制草案 → 服务校验 → 影响预览 → 发布地理投影版本”的最小完整闭环。

## 任务清单

- [x] 固定广州受控范围、双模式长期保留和切换不重置业务数据的合同。
- [x] 定义运营空间方案、空间要素、版本和生命周期边界。
- [x] 注册运营空间方案字段、状态和目标对象类型的统一中文合同。
- [x] 建立方案服务：创建草案、几何校验、影响预览、发布和取消。
- [x] 接入成熟地图绘制适配器，不在页面手写多边形交互。
- [x] 在地理地图直接提供“规划运营空间”入口；网格模式不显示编辑能力。
- [x] 支持 Zone、Place、ServiceArea 选择、绘制、撤销、重做和取消。
- [x] 已发布方案覆盖地理投影；旧网格空间、业务对象和模拟运行保持不变。
- [x] 对静态图层和动态 Robotaxi 图层实施差异更新，避免地图重复初始化。
- [x] 完成服务合同、投影、桌面、手机、旧网格和真实页面验证。
- [x] 更新版本、用户更新记录、提交和本地标签；公网发布仍由用户决定。

## 验收闭环

1. 进入中控台仍可在“地理 / 网格”之间切换，网格地图与既有交互不回归。
2. 地理模式可直接进入规划，选择对象类型和目标对象后在地图绘制多边形。
3. 非法几何、广州范围外几何或缺少目标信息不能发布，并显示中文原因。
4. 校验通过后可查看影响并发布；已发布版本刷新后仍存在。
5. 发布只改变地理投影，不把新草案加入模拟运行，不改写历史 Route 和行驶事实。
6. 地图缩放、移动和 Robotaxi 更新不会重复创建地图实例或全量刷新无关图层。
