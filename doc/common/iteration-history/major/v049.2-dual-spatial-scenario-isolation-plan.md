# v049.2.0 双空间场景隔离实施计划

状态：已完成

## 目标

把当前“网格对象投影到广州底图”的混合实现拆分为独立的 `城市地理` 与 `网格仿真` 场景。统一对象定义和能力接口，隔离空间数据、道路、位置、路线和运行资格，保证现有业务闭环不回归。

## 本版本边界

- 网格仿真继续作为当前业务运行场景。
- 城市地理成为独立的广州运营空间规划场景。
- 城市地理不显示网格 Cell、模拟道路、网格 Robotaxi 或网格 Route。
- 本版本不宣称完成真实道路路由和城市 Robotaxi 运行；后续道路图和 Routing Provider 通过能力门禁接入。

## 验收标准

1. 城市地理中不存在 `cell_id` 派生道路、车辆或路线图层。
2. 网格仿真对象、路径规划、单据和模拟运行结果保持不变。
3. 城市方案不能覆盖网格场景，其他场景方案不能污染广州场景。
4. 页面明确显示当前查看场景及当前业务运行场景能力。
5. 城市地理场景生成性能稳定，切换场景不重建业务数据。
6. 桌面和手机均可查看、切换和进行已有空间规划操作。

## 后续入口

`城市地理` 只有在广州道路图、城市路由 Provider、上下车点和地理位置 Provider 全部完成后，才能进入“业务运行场景启用”迭代。该动作必须新建运行快照或显式迁移，禁止由地图切换按钮触发。
