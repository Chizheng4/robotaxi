# v041.4 手机响应式基础与版本化发布计划

## 状态

状态：已完成。

## 目标

建立平台级移动端响应式基础，使登录、键盘输入、导航、工作区、详情和浮层在手机上具备基本可用性；同时将线上发布改为版本标签驱动，确保 GitHub Actions 和线上构建明确显示 Git 版本号。

本轮继续以桌面运营工作台为主，不把所有复杂表格改造成手机 App，也不改变业务对象、单据服务、状态机或模拟运行。

## 用户体验原则

1. 响应式规则服务用户理解和操作，不机械压缩桌面内容。
2. 手机端优先保证内容完整、控件可达、滚动边界清楚和当前操作可恢复。
3. 键盘、地址栏、横竖屏导致的视口变化由统一能力处理，不由单页写补丁。
4. 高密度表格允许横向浏览，避免为了“塞进去”而隐藏业务字段。

## 任务清单

### 任务 1：统一可视视口服务

- [x] 新增响应式视口服务，统一输出可视高度、键盘占用、安全区和设备断点。
- [x] 使用 `visualViewport` 与稳定高度判断软键盘，不依赖固定设备型号。
- [x] 通过根 CSS 变量和数据属性服务登录页、平台框架、详情和浮层。

### 任务 2：登录与软键盘

- [x] 标题、输入框、按钮和错误反馈在键盘打开时保持完整可见。

### 任务 3：手机平台框架

- [x] 手机端统一系统栏、导航、页签、主内容和详情的高度及滚动职责。

- [x] 手机顶部保留页面标题、版本和用户入口，弱化非核心摘要操作。
- [x] 详情在手机端使用完整可读宽度，表格和筛选保留横向浏览能力。

## 验收标准

1. 手机键盘打开后登录标题、输入框和按钮不被裁切，页面可自然滚动。
2. 页面框架高度跟随当前可视区域，不出现全局横向溢出和不可恢复裁切。
3. 手机端核心导航、详情和浮层可打开、关闭和滚动。
4. 桌面端现有业务页面、地图、详情和模拟运行保持正常。
5. 推送 `main --follow-tags` 后，GitHub Actions 标题直接显示 `v041.4.0`。
