# v045 经营模型驱动的规划与分析重构计划

## 1. 目标

建立经营模型作为经营规划和经营分析的共同合同，统一指标语义、规划输入、执行结果、经营事实和偏差反馈；同时抽离导航注册表并统一桌面、手机分析画布。

## 2. 边界

- 不修改业务单据生命周期、财务事实归属或模拟运行主路径；
- 菜单顺序调整只改变信息架构；
- 指标定义继续作为语义合同，计算记录继续作为审计事实；
- 前端不新建第二套计算逻辑。

## 3. 任务

- [x] 更新经营模型、经营规划和经营分析正式设计；
- [x] 建立 `OperatingModelDefinition` 服务、模型域和关系验证；
- [x] 同步字段字典和中文展示合同；
- [x] 抽离导航注册表、页面路径查询和唯一性验证；
- [x] 一级菜单调整为中控台、经营规划、经营分析、客户、供应、Robotaxi、运营、财务、组织、地图、模拟；
- [x] 新增经营模型结构视图，不复制计算或业务事实；
- [x] 统一经营规划与经营分析的分析画布和移动端规则；
- [x] 完成桌面、手机、浏览器、业务合同、模拟运行和性能验证；
- [x] 更新版本记录、归档计划并提交 `v045.0.0`。

## 4. 验收

1. 经营模型能解释需求、供给、服务、资产、财务和反馈关系；
2. 规划与分析引用同一模型和指标语义；
3. 菜单排序只修改导航注册表；
4. 经营分析手机端无页面级横向溢出，图表和卡片可读；
5. 模拟运行和既有业务服务合同全部通过。
