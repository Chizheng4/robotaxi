# v045.3.0 决策控制中心迭代归档

## 1. 版本目标

建立连接经营规划与业务执行的决策控制层，统一观察跨价值流策略、执行、结果、异常和经营效果，同时保持每个策略对象及其执行闭环由原业务服务独立拥有。

## 2. 架构边界

- 决策中心不是第二套策略数据源，不创建重复的策略、执行或结果对象。
- 策略配置、执行和结果继续归属供应、供需投放、出行服务、运维支持、Robotaxi 和运营模拟等价值流。
- 决策中心通过能力目录和只读投影统一展示；经营效果指标由统一经营数据池计算并引用。
- 页面只展示和导航，不在页面中统计策略事实或拼装决策闭环。
- 本版本不改变业务单据生命周期，不新增模拟运行扫描和逐 Tick 计算。

## 4. 验收结果

1. 任一策略事实只有一个业务归属，决策中心不复制或修改原对象。
2. 决策中心统一展示策略能力、执行次数、成功率、结果覆盖、异常和关联经营指标。
3. 决策过程指标只由统一经营数据池提供，页面不重复计算。
4. `供需投放`、`出行服务`、`运维支持`成为一级价值流；虚拟需求策略归入运营模拟。
5. 决策中心使用分析页面合同，无右侧详情、搜索表单和最近任务事件。
6. 桌面端与 390×844 手机视口通过真实浏览器验证，页面无整体横向溢出和控制台错误。
7. 业务单据生命周期与模拟运行主路径未变更。
