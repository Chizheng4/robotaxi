# v036 模拟运行排空与运行队列大版本计划

## 状态

v036.1 已完成。本轮目标是修复“计划 Tick 结束但当前运行仍有时间作业未完成”的根因，并继续降低历史诊断数据对当前模拟运行的影响。

## 背景

v035 已完成当前运行作用域、时间作业候选推进、运行态缓冲刷新和空闲跳时。但仍存在三个关键风险：

1. `DRAINING` 收尾完成判断主要依赖工作流动作数量，未把当前运行未完成时间作业作为运行未完成条件。
2. 时间作业仍是数组诊断数据上的候选过滤，不是严格的当前运行调度队列。
3. 成功终态事件和时间作业长期保留会增加前端展示态、持久化和派生数据压力。

## 目标

1. 模拟运行真正完成必须满足：当前运行无活跃业务对象、无待执行工作流动作、无未完成时间作业。
2. 计划 Tick 结束只表示停止新需求/供给触发，不表示运行完成。
3. 收尾阶段必须继续按统一模拟时间推进时间作业，可跳到下一个到期时间作业秒。
4. 当前运行热路径不得被历史成功诊断数据拖慢；失败、取消、异常数据必须保留用于排查。
5. 删除过时的当前迭代待定指标文件，避免后续模型误判当前任务。

## v036.1 执行范围

状态：已完成。

1. `executeTick` 返回当前运行活跃时间作业数量和活跃业务对象数量。
2. `completeTick` 的 `DRAINING` 完成条件增加活跃时间作业和活跃业务对象约束。
3. `resolveIdleJumpTargetSeconds` 在 `DRAINING` 阶段只考虑时间作业、行驶快照和排空边界，不再被需求/供给周期干扰。
4. 新增运行时诊断清理能力：保留当前运行活跃作业、失败/取消作业，压缩历史成功终态作业。
5. 模拟事件保留继续遵循上限，失败事件优先保留。
6. 删除 `doc/common/current-iteration/major/operating-metrics-model-pending-plan.md`。
7. 更新规则和验证，确保排空不丢时间作业、历史成功诊断不会拖慢当前运行。

## 验收

1. 当前运行存在未完成时间作业时，`DRAINING` 不得完成。
2. 未完成时间作业到期后，仍通过原业务动作服务执行。
3. `DRAINING` 跳时优先跳到下一时间作业或行驶快照，不受需求/供给触发周期影响。
4. 成功终态历史时间作业可被清理，失败/取消作业保留。
5. 当前迭代目录不再保留过时待定指标方案。
6. 提交前检查通过，完成后归档本计划并打 tag。
