# v043.0 长期需求预测与 Robotaxi 容量规划重构计划

状态：已完成

## 1. 版本目标

将现有演示性需求预测器重构为确定性、参数化、可解释、可复盘的长期经营容量规划服务，完整打通：

```text
经营目标 + Place 需求画像 + ServiceArea 承载画像
+ Zone 汇总 + Robotaxi 资产与能力 + 生产画像
→ 预测策略 → 预测执行 → 预测结果 → 生产计划输入
```

本版本不接入模拟运行主路径，不实现机器学习、随机预测、实时调度或完整财务模型。

## 2. 已确认决策

1. Place 是需求唯一来源；ServiceArea 只表达承载能力且唯一归属 Place；Zone 只汇总。
2. 支持周、月、季度、年预测周期，增长率单位必须与预测周期一致。
3. 市场预测、经营目标、服务承载和供给能力分别计算，不互相覆盖事实。
4. Robotaxi 需求同时计算日订单能力、峰值并发和经营最低规模。
5. 所有计算数量字段统一使用 `robotaxi`，废弃 `vehicle_*` 和模糊的 `fleet` 计算口径。
6. 生产、检验、爬坡、交付均形成真实时间和数量约束。
7. 每次执行冻结完整输入快照；历史结果不可变。
8. 预测结果使用分析型页面，计算过程可以展开复盘。
9. 固定置信度废弃，以数据质量和缺失输入替代。
10. 预测结果可生成生产计划，但两个对象保持独立服务边界。

## 3. 当前根源差异

|范围|当前问题|目标状态|
|---|---|---|
|需求画像|需求权重重复、人口直接相加、ServiceArea 转换需求|人群权重分层，Place 生成需求，ServiceArea 只校验容量|
|预测周期|固定年度且混用目标、策略、生产周期|单一周期单位和数量，生产提前期不改变市场周期|
|增长|优先复用画像旧增长因子|执行按 Place 加权增长率和策略情景实时计算|
|经营目标|累计订单除以天数，目标来源难解释|直接使用期末日订单，并输出市场机会和目标不足|
|Robotaxi 能力|只使用载客时长，字段仍为 vehicle/fleet|完整服务周期、可用率、利用率、日常与峰值拆解|
|区域资产|`Z-001` 特殊统计全部车辆|按真实 Zone、交付、准入、调入调出和退役统计|
|生产约束|交付能力不能真正限制结果|自然周期累计生产、检验、爬坡和交付约束|
|执行复盘|快照不完整|冻结全部输入、参数、假设与校验|
|结果质量|固定 `confidence_level = 0.82`|数据质量、缺失字段和默认假设|
|前端|普通表格与详情堆字段|分析结论、趋势、瓶颈、规模拆解、时间线和计算过程|

## 6. 风险控制

1. 旧字段兼容只能在加载边界存在，禁止进入新计算内核。
2. 画像整体重算不能修改 Place、ServiceArea、Zone 的物理事实。
3. 新预测对象默认不进入模拟运行扫描。
4. 生产计划只消费已生成结果，不重复预测计算。
5. 每个子版本只完成一个稳定合同，验证通过后再进入下一阶段。
6. 若发现 ServiceArea 现有多 Place 关系被其他业务使用，只迁移经营画像归属，不直接删除物理关系字段，先建立兼容读取。
