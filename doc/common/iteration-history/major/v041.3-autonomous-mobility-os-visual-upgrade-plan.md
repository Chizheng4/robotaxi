# v041.3 Autonomous Mobility OS 视觉升级计划

## 状态

状态：已完成。

## 目标

将 Robotaxi 运营平台从传统 SaaS 工作台视觉升级为“管理一个持续运行的自动驾驶交通网络”的 Autonomous Mobility Operating System，同时保持现有业务对象、单据服务、状态机、模拟运行和数据持久化不变。

## 架构边界

1. 全局工作台视觉升级不改变菜单层级、页面编号、工作台页签和页面状态结构。
2. 运营中控台使用地图优先和对象优先表达；经营规划、财务分析、策略和业务单据继续使用适合自身的分析页或紧凑表格页。
3. Robotaxi、运营中心、服务区域和空间对象只消费现有服务化数据，不在页面创建第二套对象或运行态。
4. 地图选择对象继续调用现有 `selected`、详情和业务操作链路。
5. 登录、用户和版本记录只属于平台体验层，不写入业务运行态，不接入模拟运行。
6. 不新增业务字段、状态或枚举；如实现中发现确需新增，必须暂停并先同步字段字典。

1. 顶部系统栏视觉连续，左上品牌不再形成独立标题块。
2. 左侧导航具备概念稿的轻量质感，同时完整承载当前多层菜单。
3. 登录页控件纵向等宽并适度上移，桌面和手机均无裁切。
4. 运营中控台第一视觉为真实业务地图和对象，而不是卡片或表格。
5. Robotaxi 地图对象来自当前数据并可进入现有详情链路。
6. 业务页面和模拟运行回归无异常，工作区无未提交文件。
