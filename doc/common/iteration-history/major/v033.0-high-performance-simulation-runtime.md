# v033.0 大版本：高性能模拟运行内核与扩展规范

## 版本类型

大版本。

## 背景

当前系统已经建立统一模拟时间、时间作业和工作流自动执行，但高速运行时出现明显卡顿。根因不是单个页面样式问题，而是模拟循环把每个 1 秒世界 Tick 都转化为前端事件、React state 写入、全量业务对象扫描和持久化压力。

随着后续业务单据、状态节点、策略、成本、收入和指标体系继续扩展，如果不建立统一模拟运行架构规则，每次业务扩展都可能再次绕开底座形成补丁式逻辑。

## 目标

1. 建立长期模拟运行架构规则，约束未来所有模型和开发者。
2. 保持 1 秒世界时间精度，但避免空 Tick 事件洪流。
3. 降低高速模拟对 React 渲染、IndexedDB 持久化和调试日志的压力。
4. 保持业务闭环唯一来源：模拟只调度，业务动作仍由原业务服务执行。
5. 为后续事件驱动跳时、活跃对象索引和时间作业调度队列留下结构入口。

## 第一阶段执行范围

- 新增 `doc/rules/07-simulation-runtime-architecture-rules.md`。
- 更新 `AGENTS.md` 和 `doc/iteration-rules.md`，让后续模拟相关迭代强制读取新规则。
- 新增模拟性能配置，控制事件记录、检查点、前端快照、内存事件上限和持久化防抖。
- 高速模拟路径下不再记录每个 Tick 的开始/结束事件，只记录业务事件、异常和检查点。
- 调试日志默认关闭并设置上限。
- 时间作业 state 写入改为有变化才写入。
- 前端运行快照持久化增加防抖。

## 不做范围

- 本阶段不一次性完成完整事件驱动跳时内核。
- 本阶段不重写所有业务对象为 Map 索引。
- 本阶段不进入指标体系。

## 验收

- 提交前检查通过。
- 页面入口正常加载。
- 新增验证覆盖性能配置、事件节流、调试日志上限和时间作业无变化不写入。
