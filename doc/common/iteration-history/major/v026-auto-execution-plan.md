# v026 自动执行计划：全站视觉体验升级

## 1. 版本目标

以已确认的方案 3「聚焦型运营工作台」作为全站视觉基线，并将方案 2「地图运营中控台」用于地图场景，建立简洁、克制、优雅且适合高频运营工作的 Robotaxi 前端体验。

本版本只调整前端视觉系统、布局表达与交互反馈，不修改业务对象、字段、状态、枚举和业务流程。

## 2. 设计原则

1. 保留 B 端运营系统的信息密度，不改造成营销页或卡片式大屏。
2. 使用中性灰白表面、近黑文字、细分隔线和克制蓝色强调，降低默认组件拼装感。
3. 通过字体、间距、对齐和状态表达建立层级，减少重复标题、厚重边框和无效留白。
4. 列表页保持状态分类、搜索过滤、全局操作、表格、详情与底部信息闭环。
5. 地图页保持地图主画布优先，强化道路、地点、服务区、路线、节点和图例的辨识度。
6. 不覆盖用户未提交修改，不提交无关指标文档。

## 3. 自动执行范围

### v026.1 视觉基础与全局框架

- 重构全局颜色、字体、间距、边框、圆角、阴影和交互状态。
- 优化侧边导航、品牌入口、顶部信息区和工作台页签。
- 统一按钮、输入框、选择器、状态标签和滚动条体验。

### v026.2 业务列表与详情体验

- 优化状态分类、筛选区、操作区、表格行、分页和底部信息区。
- 强化当前行、悬停行、关键状态和固定操作列的视觉表达。
- 优化右侧详情栏、空状态、详情 Tab 和字段阅读节奏。

### v026.3 地图与模拟运行体验

- 优化地图画布、地图工具、道路和区域颜色、选中路线与图例。
- 优化模拟运行表格、事件区、拖动分隔条和运行状态信息层级。
- 确保地图、表格、事件区和详情区边界对齐。

### v026.4 前端设计系统标准化

- 建立模型无关的前端设计系统规则，定义视觉原则、设计令牌、页面模板、组件状态和验收标准。
- 将设计系统加入 `AGENTS.md` 强制读取与完成自检，确保 Codex 或其他模型执行前端任务时使用同一标准。
- 规定新增模块、局部迭代和跨模块改造的统一接入流程，禁止一次性视觉规则。

### v026.5 验证与归档

- 重新构建 `src/main.bundle.js` 并执行语法检查。
- 重启本地服务，验证页面正常加载。
- 模拟真实运营路径，覆盖菜单、状态分类、查询、重置、表格选择、详情折叠、地图和模拟运行页面。
- 以选定效果图为视觉基准完成设计 QA，修复 P0/P1/P2 问题。
- 更新 `VERSION.md`，归档本计划，提交并创建 `v026` 标签。

## 4. 验收标准

1. 全站视觉语言统一，首屏不再呈现明显默认组件拼装感。
2. 侧边导航、顶部栏、页签、主内容和详情栏层级清晰且边界对齐。
3. 列表页完整保留状态分类、查询重置、操作、表格、详情与分页结构。
4. 所有可见业务文案继续使用中文，不直接暴露内部字段名或英文枚举值。
5. 地图主体获得最大可用空间，地图要素、工具和图例易于识别。
6. 1280 x 720 与 1440 x 900 桌面视口下无文字遮挡、布局错位或不可操作区域。
7. 核心交互和运行态持久化保持可用，前端无新增控制台错误。
8. `design-qa.md` 最终结果为 `passed`。

## 5. 风险控制

- 视觉升级不改变业务逻辑，避免影响已跑通的自动运营模拟闭环。
- 优先复用现有 Ant Design 组件和项目结构，不引入新的运行时依赖。
- 当前 `index.html` 的缓存版本号属于用户未提交修改，构建时只在其基础上更新，不回退历史值。
- `doc/06-metrics/` 当前为空且与本版本无关，不纳入提交。
