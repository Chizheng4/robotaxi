# v034 经营指标系统大版本计划

## 状态

v034.5 已完成。当前已建立指标定义、指标计算引擎、经营指标页面、过程诊断和自动计算验证闭环。

## 目标

基于现有模拟业务闭环、成本事实和收入事实，建立经营指标系统，使平台能够从模拟运行结果中生成可解释、可下钻、可复盘的经营指标。

## 第一版 P0 指标

|指标|来源|页面|
|---|---|---|
|应收收入|RevenueRecord|经营指标总览、财务指标|
|实收收入|RevenueRecord|经营指标总览、财务指标|
|未收收入|RevenueRecord|财务指标|
|运营总成本|CostRecord|经营指标总览、财务指标|
|贡献利润|RevenueRecord - CostRecord|经营指标总览、财务指标|
|贡献利润率|贡献利润 / 实收收入|财务指标|
|创建订单数|ServiceOrder|服务指标|
|完成订单数|ServiceOrder|经营指标总览、服务指标|
|订单履约率|完成订单数 / 创建订单数|经营指标总览、服务指标|
|订单取消率|取消订单数 / 创建订单数|服务指标|
|单均收入|实收收入 / 完成订单数|财务指标|
|单均成本|运营总成本 / 完成订单数|经营指标总览、财务指标|
|关键数据完整率|模拟时间、成本、收入、状态时间线|质量护栏|

## 前端体验要求

1. 不做营销式大屏，不使用大标题、厚重色块或装饰背景。
2. 指标总览必须信息密度高但层次清晰。
3. 指标卡片用于表达单个核心指标，避免卡片套卡片。
4. 趋势、构成、异常和详情必须能互相解释。
5. 所有指标名称、字段名、状态和枚举必须来自统一字段字典。
6. 指标详情必须展示：定义、公式、来源字段、窗口、质量状态、来源对象。

## 执行原则

1. 先确认方案，再编码。
2. 每个子版本独立提交和打标签。
3. 任何新增指标对象、字段、枚举必须同步字段字典文档和前端字段字典。
4. 前端必须复用经营分析管理的统一工作台骨架。
5. 指标计算只消费事实，不修改业务单据生命周期。
6. 不进入预测模型、潜在需求、车辆状态历史快照等 P1/P2 能力，除非用户明确确认。

## 暂停条件

进入编码后如出现以下情况必须暂停确认：

1. 指标字段命名存在多种合理选择。
2. 成本或收入事实与业务单据对账不一致。
3. 指标页面需要新增超出经营分析管理的导航结构。
4. 自动计算触发会造成性能风险。
5. 需要新增车辆状态历史快照、潜在需求事实等 P1 能力。
