# v044 经营规划与经营分析一体化自动执行计划

状态：执行中

## 1. 版本目标

建立统一经营数据池和唯一指标语义，以经营规划为基线、业务事实为实际结果，形成“目标—预测—执行—事实—偏差—反馈”的经营管理闭环；同时统一经营分析画布、地图摘要和页面上下文。

本版本不重写业务单据，不改变模拟运行主路径，不引入服务器数据库或大型图表依赖。

## 2. 已确认原则

1. 业务单据生命周期是底层事实，模拟运行是上层扩展。
2. 经营数据只能由统一数据池服务聚合和提供，页面不得各自计算。
3. 一个指标只有一个编号、中文名、含义、单位、公式和时间口径。
4. 经营分析必须同时表达目标值、预测值、实际值、差异和达成率。
5. 经营分析是全宽分析画布，不继承常驻详情和最近任务事件。
6. 地图摘要从对象和统一数据池读取，桌面和手机共用响应式摘要组件。
7. 页面名称和说明由统一页面上下文注册表提供。

## 5. 风险控制

1. 数据池只引用事实，不复制或覆盖业务单据。
2. 旧指标编号兼容只存在于读取边界，新页面只使用唯一指标语义。
3. 规划期末目标与当前同期预测分开表达。
4. 新计算不进入模拟 Tick，只在统一经营数据更新时执行。
5. 地图悬停只读取预建索引，不触发经营数据重新计算。
