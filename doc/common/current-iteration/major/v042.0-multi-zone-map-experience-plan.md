# v042.0 多区域运营空间与地图体验升级计划

## 状态

状态：执行中。

## 目标

将运营中控台升级为可平移、缩放、识别真实运营对象的全幅地图场景；在不改变 `Z-001` 现有 ID、坐标、关系和业务结果的前提下，增加规划态 `Z-002` 完整空间结构，为未来双区域运营复制建立真实对象基础。

## 架构边界

1. `Z-001` 继续是当前唯一运营作用域，现有 Robotaxi、订单、任务、路线、服务区域和运营中心逻辑保持不变。
2. `Z-002` 是真实空间对象集合，但当前为规划态；道路、地点、服务区域和运营中心不得进入需求生成、车辆匹配、路径规划、运维调度和模拟运行。
3. 地图展示消费空间目录和画像数据，只做投影、交互和详情触发，不修改业务对象。
4. 旧快照通过空间目录合并服务补充新增静态对象，不覆盖已有业务运行态。
5. 地图移动和缩放属于纯前端视口状态，不写入业务运行态，不触发模拟计算。
6. 地图不得维护对象信息副本；名称、类型、状态、位置和画像值必须从当前系统对象及统一画像服务派生。展示注册表只定义字段选择、层级和视觉语义，不保存业务值。

## 任务清单

### 任务 1：兼容基线与空间目录

- [ ] 固化 `Z-001` ID、坐标、对象数量、路线端点和默认业务策略目标区域基线。
- [ ] 建立空间对象运营作用域判断，规划态对象不进入业务候选集合。
- [ ] 建立版本化空间目录合并能力，旧快照自动补充新增空间对象。

### 任务 2：Zone 2 空间结构

- [ ] 地图扩展为横向双区域结构，`Z-001` 原坐标保持不变。
- [ ] 新增 `Z-002`、子区域、道路、道路节点、地点和服务区域。
- [ ] 新增跨区域连接走廊，但当前不生成跨区域运营路线。
- [ ] 为 Zone 2 生成规划画像，保留未来区域启用合同。

### 任务 3：高性能地图场景

- [ ] 地图占满工作台可用区域，移除固定底部图例和常驻统计遮挡。
- [ ] 网格、道路、地点、服务区域和 Zone 使用合并几何图层，减少 DOM 节点。
- [ ] 平移和缩放通过视口控制器与 `requestAnimationFrame` 推进，不在每次指针移动时重绘业务对象。
- [ ] 支持鼠标滚轮、按钮、拖动、触控和复位，并约束有效视口范围。

### 任务 4：对象信息体验

- [ ] 按缩放层级显示 Zone、Place、服务区域、道路、运营中心和 Robotaxi 标签。
- [ ] 建立地图对象展示注册表，只配置对象字段选择、中文展示、图标、颜色和缩放层级；字段值实时来自当前对象和画像服务。
- [ ] 建立地图展示选择器，对象或画像更新后自动重新派生对应地图投影，不增加页面同步动作。
- [ ] 鼠标悬停显示最多四项关键画像；点击后关闭悬停并进入现有右侧详情。
- [ ] 手机无悬停时直接点击对象查看详情。
- [ ] 图层说明改为按需浮层，不长期遮挡地图。

### 任务 5：验证与收口

- [ ] Zone 1 初始化和模拟业务结果与升级前基线一致。
- [ ] Zone 2 规划对象不进入需求、匹配、路径、运维和模拟候选。
- [ ] 旧快照、空快照、重置数据均可加载双区域地图。
- [ ] 地图空间 DOM 节点控制在 500 个以内，10,000 辆车投影继续保持毫秒级。
- [ ] 验证 `1280×720`、`1440×900`、手机视口和详情展开状态。
- [ ] 更新字段字典、空间设计文档、版本记录和网站用户更新记录。
- [ ] 完成完整提交前检查、归档、`v042.0.0` 提交、标签和公网发布。

## 视觉验收

1. 地图是工作台主场景，不表现为卡片或被固定说明条切割。
2. 双 Zone 横向构图充分利用宽屏，详情展开和收起时都能自然适配。
3. 低缩放只显示区域层级，高缩放逐步展示道路和运营对象，避免标签堆叠。
4. 悬停信息轻量、短暂、靠近对象；完整信息只在右侧详情呈现。
5. 所有对象名称、状态和画像字段使用统一中文展示服务。

## 性能预算

- 静态空间几何只在空间目录变化时计算。
- 指针拖动每帧不写 React 业务状态。
- 地图空间静态 DOM 节点不随 Cell 数线性增长。
- 单帧视口更新目标不超过 16.7ms。
- 地图交互不触发运行态持久化。
