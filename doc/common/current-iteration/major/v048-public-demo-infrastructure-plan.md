# v048.0.0 公开演示基础设施与访问记录

## 价值目标

在不改变 Robotaxi 业务事实和 GitHub Pages 公开入口的前提下，为作品演示建立轻量、可持续保留、仅本人可查看的匿名访问反馈，让公开传播能够形成真实但克制的使用证据。

## 任务清单

- [x] 固定 GitHub Pages 前端与 EdgeOne API/KV 的职责边界。
- [x] 登录页增加“访问”专用入口，每次进入必须验证密码。
- [x] 建立独立访问会话、可见时长、来源、设备、版本和匿名访客记录。
- [x] 建立 HMAC 匿名化、来源白名单、短期内存令牌、输入限制和降级合同。
- [x] 建立手机与桌面共用的轻量访问记录页面。
- [x] 接入统一字段字典和中文展示。
- [x] 增加 EdgeOne Functions、KV 聚合和本地自动验证。
- [x] 增加仅限本机域名的本地存储适配器，审核期间可验证完整交互且不污染正式站数据。
- [x] 完成 `v048.0.0` 接入准备版本记录、完整门禁、稳定提交和标签。
- [ ] KV 审核通过后完成命名空间、环境变量和正式 API 域名绑定。
- [ ] 在 EdgeOne 真实环境验证写入、密码、周期查询和公网跨域。
- [ ] 完成访问记录正式线上发布和公网验收。

## 阻塞边界

代码、文档和本地合同可以在 KV 审核期间完成。只有 EdgeOne 真实 KV 写入、公网 API 域名和端到端线上验收必须等待审核；未完成这些步骤前不得声称线上访问记录已经启用。
