# v035 模拟运行性能架构大版本计划

## 状态

v035.1 已完成。当前已把模拟运行从“全量历史对象逐 Tick 扫描”收敛为“当前运行活跃对象 + 活跃时间作业”的运行时作用域，保证连续多次模拟运行不会因为历史业务数据累计而显著变慢。

## 背景

当前模拟运行已经具备统一世界时间、工作流时效配置、时间作业、业务动作同源执行和高速 Tick 配置。但运行时仍存在性能根因：

1. 每个 Tick 查询工作流时扫描全量历史业务对象。
2. 多个已完成 SimulationRun 产生的历史订单、任务、履约和行驶记录仍会进入后续运行扫描。
3. 时间作业和行驶进度推进仍存在全量集合处理倾向。
4. React state 仍承载较多逐 Tick 更新，长期应继续向运行态内核和阶段性快照迁移。

## 时间语义边界

1. `tick_seconds = 1` 是统一世界时间精度，不等于所有业务每秒触发。
2. 模拟加速只改变真实等待时间，不改变模拟世界里的业务耗时。
3. 业务动作耗时必须来自工作流时效配置或路径行驶计算。
4. 需求触发、供给触发、时间作业到期、工作时间边界是未来事件驱动跳时的候选时间点。
5. 如果某动作配置为 10 秒，则状态变更必须发生在模拟时间 `T + 10s`，不能因为真实执行加速而缩短，也不能因为需求触发周期而粗粒度延后。

## 版本拆分

### v035.1 当前运行作用域与活跃对象扫描

状态：已完成。

范围：

1. 新增模拟运行作用域服务，统一筛选当前 SimulationRun 相关的活跃业务对象。
2. 工作流查询只消费当前运行活跃对象，历史运行对象和终态对象不进入自动推进扫描。
3. 时间作业推进只面向当前运行的非终态作业，仍按模拟世界时间精确到期。
4. 动作执行仍使用完整业务数据上下文，保证路径规划、匹配、结算、支付等原业务闭环不被替换。
5. 新增合同验证，覆盖历史运行终态对象不被扫描、当前运行待处理对象继续推进、业务动作耗时语义不改变。

### v035.2 活跃业务索引服务化

状态：待执行。

范围：

1. 将活跃对象规则扩展为可配置接入合同。
2. 新业务对象接入模拟时必须声明状态字段、终态集合、主键字段和是否参与活跃扫描。
3. 逐步用索引替代多数组全量过滤。

### v035.3 时间作业到期索引

状态：待执行。

范围：

1. 时间作业按 `simulation_run_id + operation_status + planned_completed_seconds` 建立到期候选集合。
2. Tick 内只处理当前运行已到期和需要 UI 快照的行驶作业。
3. 保留时间作业明细页面的核查能力，不以页面展示驱动运行时扫描。

### v035.4 展示态节流与运行态分离

状态：待执行。

范围：

1. 进一步降低 Tick 批次内 React setState 频率。
2. 运行态内核在普通 JS 对象中推进，React 仅接收阶段性快照、业务事件和必要进度。
3. 持久化、校验和表格派生继续保持节流。

### v035.5 事件驱动跳时

状态：待执行。

范围：

1. 在无业务事件的时间段直接跳到下一个业务相关时间点。
2. 下一个时间点必须在需求触发、供给触发、时间作业到期、工作时间边界、模拟结束边界中取最小值。
3. 跳时不得跳过任何业务动作配置耗时或状态时间线记录。

## 工程原则

1. 正确性优先：模拟必须继续调用原业务动作闭环。
2. 作用域收敛：当前运行不扫描历史运行无关对象。
3. 活跃优先：终态对象退出自动推进扫描。
4. 调度优先：时间作业按到期推进，不做无意义全量重写。
5. 展示节流：高速运行不让 React 每 Tick 承担全量计算。
6. 扩展合同化：新增业务必须声明模拟接入合同。

## 验收

1. 多个已完成模拟运行存在时，新运行只扫描当前运行活跃对象。
2. 当前运行的非终态服务订单、任务、行驶记录和履约记录仍可继续自动推进。
3. 工作流时效配置、路径行驶耗时和订单等待耗时不被改变。
4. 事件数量继续受上限控制，不记录 Tick 洪流。
5. 提交前检查通过。
