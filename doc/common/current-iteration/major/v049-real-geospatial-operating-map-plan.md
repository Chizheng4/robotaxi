# v049.0.0 真实地理空间运营地图升级实施计划

## 状态

- 状态：执行中。
- 版本边界：`v048` 工程阶段已经完成；访问记录已在 `v049.2.1` 迁移到 CloudBase，控制台部署和公网验收保留为独立外部待办，不阻塞地图工程实施。
- 正式方案：`doc/01-space-model/12-real-geospatial-operating-map-design.md`。
- 当前中版本：`v049.6.0` 地图驱动城市空间规划；城市正式目录从空状态开始，地图视角约束规划对象类型，几何服务自动推断空间关系，并统一桌面悬停与手机轻触摘要，不接管网格业务运行。

## 价值目标

在不破坏现有模拟网格、空间对象、路径规划服务、行驶记录和模拟运行的前提下，引入真实地理坐标、可治理几何、矢量地图和真实道路路由，使地图成为空间经营对象的统一可视化与操作入口，并跑通从空间对象、路径规划到业务单据到达结果的完整闭环。

## 不变约束

1. Place 产生需求，ServiceArea 表达服务承载，Zone 只聚合；地图升级不得改变经营语义。
2. Route 是路径规划结果，不是地图静态初始化对象。
3. 页面只展示和触发；空间对象、路径规划、行驶和单据状态全部通过服务层。
4. 人工操作、真实时间自动化和模拟运行复用同一业务动作服务。
5. 现有模拟网格继续可独立运行并作为回归基线。
6. 新增真实地理模式默认不进入模拟运行主扫描，完成接入合同后才能启用。
7. 历史 Route、Trip 和 RouteExecution 不因地图数据更新被改写。

## 执行约束

1. 每一阶段都必须同时有业务或数据合同、服务实现、必要字段、测试和真实页面验证，不能先用页面私有数据制造完成效果。

3. 阶段 5 只证明 Route 规划能力；阶段 6 跑通行驶记录和源单据后，才算业务可运行。
4. 阶段 7 未通过前，真实地理对象不得加入模拟运行主扫描。
5. 所有候选字段在正式实施前进入统一字段字典评审；本计划本身不提前注册未实现字段。
6. 每阶段完成后更新任务状态和验证证据，不允许在版本结束时一次性补记全部完成。

## 暂停条件

出现以下情况必须暂停执行并重新确认：

1. 真实地图要求改变 Place、ServiceArea 或 Zone 的既有经营语义；
2. 第三方地图或路由许可不允许当前公开演示方式；
3. 国内地图坐标或数据要求无法在当前免费方案下明确满足；
4. 需要把真实位置、客户或订单隐私发送给第三方；
5. 路由引擎成本或部署复杂度超过当前作品演示价值；
6. 为接入真实地图需要绕过行驶记录或模拟运行服务边界；
7. 无法建立可重复的模拟网格回归基线。
