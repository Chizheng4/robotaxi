# v039 Fleet Operations 任务单状态治理与运维调度策略大版本

## 1. 目标

1. 清理所有五种运维任务单中从未使用的 CREATED 状态。
2. 补齐任务创建时的三种场景路由（Scenario 1/2/3），特别是空闲且在运维中心时直接进入 Worker 分配。
3. 新增运维调度策略模块（FleetOperationDispatchStrategy），为待分配目的地状态提供调度决策。
4. 任务单状态变更动作使用任务类型专属命名（分配清洁站、分配充电站等），统一使用 Worker 命名。
5. Robotaxi 管理页前端重组，更好展示运维容器/状态。

## 2. 子版本拆解

### v039.1: 状态清理 + Scenario 3 + 任务创建路由

- 从五种任务类型的状态枚举中移除 CREATED
- fleetOperationTaskService 增加场景 3 判断：robotaxi 空闲且在具备对应能力的运维中心 → 直接 WAITING_WORKER_ASSIGNMENT
- Robotaxi 优先级概念：高优先级任务可中断当前任务，低优先级等待
- 字段字典同步：移除 CREATED 的中文映射，新增场景相关字段

### v039.2: 运维调度策略模块

- 新增 FleetOperationDispatchStrategy / Run / Decision 三层对象
- 决策逻辑：选择最近且有空闲资源的运维中心
- 当前唯一运维中心 OC-001 具备全部能力，作为默认调度结果
- 前端页面：策略配置 + 策略执行 + 调度结果三页

### v039.3: 任务单动作接入 + 前端优化

- 五种任务单在 WAITING_DESTINATION_ASSIGNMENT 状态下接入分配动作
- 分配后触发路径规划 → 创建运营行驶记录 → 行驶推进 → 到达 → Worker 分配
- Robotaxi 管理页重组列展示：容器/状态信息前置
- Worker 命名全局统一

### v039.4: 验证 + 归档收口

- 验证脚本更新
- 提交前检查通过
- 归档
