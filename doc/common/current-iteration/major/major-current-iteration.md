# 当前大版本迭代

当前大版本：v041 供给端能力建设。

当前状态：执行中。v041 从经营规划与供给端业务底座开始，先收敛经营目标、需求画像、供给生产画像、长期需求预测，再进入自有生产、车队分配、交付、运营准入和 Robotaxi 资产进入运营的闭环。

最近完成大版本：v040 Robotaxi 运维状态治理与任务优先级调度。

## v041.1 经营规划与供给执行框架落地

目标：

1. 将需求画像收敛为经营规划层统一 `DemandProfile` 对象；
2. 使用 `target_object_type / target_object_id` 表达画像作用对象；
3. 废弃 `profile_type` 作为主口径，仅保留旧数据兼容；
4. 将经营规划层明确为经营目标、需求画像、供给生产画像、长期需求预测；
5. 将供应执行层明确为车队生产计划、生产批次、车队分配策略、交付单、运营准入；
6. 保持旧快照和现有页面可加载；
7. 不把新增经营规划和供应执行对象提前接入模拟运行主路径。

执行内容：

- 初始化模块生成统一 `demandProfiles`；
- 运行态恢复支持旧三类画像迁移；
- 前端需求画像页面展示统一目标对象字段；
- 前端需求画像页面前置展示关键配置数字和计算数字；
- 前端需求画像页面支持配置功能，按 Place、ServiceArea、Zone 类型展示不同配置字段；
- Zone 画像按包含的 Place / ServiceArea 画像汇总需求与服务容量；
- 字段字典同步统一字段和中文展示；
- 保留旧三类画像拆分作为隐藏兼容数据。
- 供应管理文档收敛为自有生产主路径，不再把车商供应、泛化供给单作为当前主线。
- 生产批次完成后通过 Robotaxi 服务创建车辆资产。
- 车队分配策略按 Zone / 运营中心分配具体 Robotaxi ID。
- Robotaxi 交付单可包含多台 Robotaxi，交付完成后逐车触发运营准入任务。

## v041.1.5 经营规划菜单小版本

目标：

1. 前端新增一级菜单“经营规划”，位置在“财务管理”之后；
2. “需求画像”从供应管理移动到经营规划下作为二级菜单；
3. 新增“供应生产画像”二级菜单；
4. 新增“需求预测”二级菜单，并按“预测策略 / 预测执行 / 预测结果”组织三级菜单；
5. 供应管理保留供应执行对象入口，不再承载需求画像和需求预测入口；
6. 本轮只做菜单、空集合占位和字段字典，不实现需求预测策略执行闭环，不接入模拟运行主路径。

后续大版本执行约束：

1. 需求预测、供应生产画像、车队生产计划、生产批次、车队分配、交付单等业务闭环进入实现前，必须先形成大版本迭代计划和任务清单；
2. 大版本计划需要明确对象、状态机、服务边界、字段字典、前端入口、验证合同和模拟运行边界；
3. 计划版本先提交，再按计划拆分小版本迭代，避免菜单、对象、服务和模拟边界混在一次修改中。

## v041.2 经营规划到供应执行自动化大版本

目标：

1. 把经营规划层从“菜单入口”升级为可复盘的业务对象闭环；
2. 让需求画像、供应生产画像和长期需求预测策略形成第一层经营规划闭环；
3. 让长期需求预测结果成为后续车队生产计划的唯一创建来源；
4. 逐步实现自有生产供给执行主路径：生产计划、生产批次、车队分配、交付单、运营准入；
5. 所有对象先作为业务底层闭环实现，不接入模拟运行主路径；
6. 页面只展示和触发服务动作，不在页面层拼业务对象或状态。

自动执行任务队列：

### v041.2.1 经营规划底座

- 初始化供应生产画像；
- 初始化长期需求预测策略；
- 实现长期需求预测策略执行服务；
- 生成预测执行记录和预测结果；
- 前端策略行操作提供“执行”；
- 运行态恢复兼容新增集合；
- 字段字典和验证合同同步。

### v041.2.2 车队生产计划

- 新增车队生产计划对象；
- 从预测结果创建生产计划；
- 计划状态机、首态时间线和确认动作；
- 前端供应管理下展示生产计划。

### v041.2.3 生产批次与 Robotaxi 资产形成

- 生产计划生成生产批次；
- 生产批次状态机；
- 批次完成后通过 Robotaxi 服务创建 N 台 Robotaxi；
- 新 Robotaxi 初始为待准入。

### v041.2.4 车队分配与交付

- 车队分配策略 / 执行 / 结果；
- 按 Zone / 运营中心输出分配数量和 Robotaxi ID 列表；
- 由分配结果创建 Robotaxi 交付单；
- 交付单支持一单多车。

### v041.2.5 交付到准入闭环验证

- 交付完成后逐车触发运营准入任务；
- 准入通过后 Robotaxi 进入可运营；
- 补齐状态时间线、事件和字段展示验证；
- 回归模拟运行不受新增业务主路径影响。

执行约束：

1. 每个子版本必须更新 `VERSION.md` 并提交 tag；
2. 每个新增对象必须同步 `doc/rules/field-dictionary.md` 和 `src/domain/fieldDictionary.js`；
3. 修改 `src/main.jsx` 必须重建 `src/main.bundle.js`；
4. 新增业务对象默认不进入模拟主路径，直到另行声明模拟接入合同；
5. 若实现过程中发现预测公式、生产计划口径或 Robotaxi 创建规则与文档冲突，必须暂停并先修正文档计划。
