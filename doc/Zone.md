## 1. 业务含义
ZONE是运营区域， 是 Robotaxi 最小运营闭环中的空间经营单元。
它定义了系统在哪个区域内进行车辆投放、需求承接、订单履约、调度优化和经营结果验证。
Zone 的核心作用不是描述地图，而是为运营系统提供一个可衡量、可优化、可复制的经营边界。
在第一阶段，Zone 用于把城市级复杂运营问题收敛为一个小区域内的供需匹配、车辆调度、服务履约和经营结果验证问题。
## 2. 关键属性

| 属性                | 含义                       |
| ----------------- | ------------------------ |
| zone_id           | 区域唯一编号                   |
| zone_name         | 区域名称                     |
| boundary          | 区域边界，用于判断车辆、需求、订单是否属于该区域 |
| zone_type         | 区域类型，例如测试区、商业区、住宅区、交通枢纽区 |
| operating_status  | 运营状态                     |
| service_policy_id | 该区域适用的服务策略               |
| map_area_id       | 对应的地图空间数据区域              |
| created_at        | 创建时间                     |
| updated_at        | 更新时间                     |

## 3. 生命周期状态

| 状态         | 含义    |
| ---------- | ----- |
| Planned    | 规划中   |
| Testing    | 测试运营中 |
| Active     | 正式运营中 |
| Restricted | 限制运营  |
| Suspended  | 暂停运营  |
| Closed     | 关闭    |

## 4. 关联说明

Zone 不直接完成调度、派单和路径规划。
它主要作为空间经营边界，被其他业务对象引用。

## 5. Zone 网络设计

在当前最小运营闭环原型中，Zone 不仅表示一个运营区域的业务边界，也承担空间网络底座的作用。为了便于模拟、观察和后续扩展，Zone 可以被离散化为由固定正方形小格组成的网格网络。每个最小网格单元具有统一边长，整个 Zone 可以配置为 N * N 的正方形网络，也可以配置为 rows * cols 的矩形网络。

这种网格化设计的目的不是还原真实地图，而是为第一阶段的运营模拟提供一个简单、可计算、可扩展的空间表达。OperatingLocation 可以落在某个网格单元上，Route 可以连接不同 OperatingLocation，后续 Vehicle、Demand、Order、Trip、DispatchTask 等对象也可以基于同一套网格空间进行位置表达、移动模拟和运营分析。

通过调整 Zone 的行数、列数和最小网格单元大小，系统可以在不接入真实地图 SDK 的情况下，逐步扩大运营区域面积、增加运营位置数量，并观察空间规模变化对供需匹配、调度和履约过程的影响。
