# 字段语义与指标血缘设计

## 1. 目标

经营规划和经营分析共用一套业务语义。字段中文名、单位、定义、所属对象、指标关系和枚举不能由页面分别维护。

```text
业务对象字段事实
  → 统一字段语义注册
  → 指标定义与计算
  → 目标值 / 预测值 / 实际值 / 差异值
  → 经营规划和经营分析统一展示
```

字段字典负责完整字段覆盖；字段语义注册负责需要跨对象、指标和页面复用的关键经营字段。两者都由字段展示服务统一提供给前端。

## 2. 唯一语义合同

关键字段至少声明：

- 唯一英文字段和中文名；
- 所属业务对象和字段性质；
- 数据类型、单位和业务定义；
- 枚举值及字段专属中文；
- 对应指标概念、实际指标和兼容字段。

只有含义、粒度、单位和公式一致时才复用字段。语义变化必须新建字段并迁移旧快照，不能只改中文后继续沿用含义模糊的旧字段。

## 3. 第一阶段指标概念

|指标概念|经营目标字段|实际指标|当前事实状态|
|---|---|---|---|
|订单履约率|`target_order_fulfillment_rate`|`OUTCOME-SERVICE-003`|可计算|
|订单服务时间利用率|`target_order_service_time_utilization_rate`|`PROCESS-ASSET-002`|缺少连续状态区间，预留|
|空驶里程占比|暂未设置目标|`PROCESS-ASSET-003`|可计算|
|载客行驶时长|`average_passenger_trip_duration_min`|暂未建立独立指标|规划可配置，实际事实待拆分|
|订单履约执行时长|`average_order_fulfillment_execution_duration_min`|`PROCESS-EFF-002`|可计算|
|订单全流程时长|`average_order_end_to_end_duration_min`|暂未建立指标|缺少完整事件时间戳时不可推算|
|车辆服务周期时长|`vehicle_service_cycle_duration_min`|暂未建立指标|规划可计算|

订单服务时间利用率只包含接驾和载客履约。剩余可运营时间可能是空闲、停车或等待，不能直接解释为空驶；空驶必须使用行驶里程事实单独计算。

## 4. 经营分析关系

经营目标保存管理意图，业务单据和 Robotaxi 保存经营事实，指标计算保存实际观测，统一经营数据池生成目标差异。经营分析不得复制经营目标或底层事实，也不得在页面临时计算。

当实际指标缺少必要事实时，目标仍可展示，但实际值和差异状态必须标记为数据不足。后续只有在 Robotaxi 连续运营状态区间、接驾区间和载客履约区间形成稳定事实后，才能启用实际订单服务时间利用率。

## 5. 订单时间语义

```text
订单创建
  → 等待匹配
  → Robotaxi 接受订单
  → 接驾
  → 乘客上车
  → 载客行驶
  → 到达下车点并送达完成
  → 车辆周转完成
```

- 载客行驶时长：乘客上车至到达下车点；
- 订单履约执行时长：Robotaxi 接受订单至送达完成；
- 订单全流程时长：订单创建至订单完成，包含等待匹配；
- 车辆服务周期时长：开始接驾至送达后周转完成，用于车辆服务产能。

四者边界不同，不得使用“履约时长”“服务周期”等模糊名称互相替代。规划参数与经营实际使用同一指标概念，但可以来自不同事实层。

## 6. 计算模型注册

跨页面计算由统一计算模型注册服务声明：模型编号、版本、输入字段、输出字段、公式、单位和业务动作。执行结果只保存模型编号、模型版本、字段键、输入值、输出值和来源引用；页面通过字段语义与模型注册解析中文名称和公式。

构建校验必须阻止以下内容：未登记输出字段、未登记计算模型、页面硬编码关键字段中文名，以及新结果继续写兼容字段。
