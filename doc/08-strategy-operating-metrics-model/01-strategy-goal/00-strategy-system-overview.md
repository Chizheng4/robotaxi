# Robotaxi 经营规划与经营反馈总览

## 1. 定位

经营规划负责定义目标、需求画像、长期需求预测和供给方向；经营分析负责用业务事实验证规划。两者共同形成经营管理闭环，但都不直接替代业务单据执行。

```text
经营目标 → 需求画像 → 需求预测 → 供给计划
   ↓                                  ↓
目标与预测基线                    业务执行层
   ↓                                  ↓
   └──────── 经营分析与偏差反馈 ← 经营事实
                         ↓
                 下一轮目标与策略调整
```

## 2. 当前已实现范围

- 经营目标、需求画像和生产画像；
- 长期需求预测策略、执行和结果；
- 预测结果生成生产计划；
- 生产批次、区域分配、交付、运营准入；
- 服务订单、行驶、Robotaxi、成本和收入事实；
- 经营指标计算与经营分析页面。

## 3. 当前升级目标

经营分析必须同时消费经营目标、预测结果、供给计划和经营事实，形成目标值、预测值、实际值、差异、达成率和驱动原因。经营规划与经营分析通过统一经营数据池关联，不通过页面相互读取，也不建立重复字段。

## 4. 系统边界

|系统|职责|
|---|---|
|经营规划|确定目标、画像、预测和供给方向|
|供应管理|把预测缺口转为生产、分配、交付和准入|
|运营管理|执行投放、服务、运维和车辆任务|
|业务事实服务|沉淀单据、状态时间线、行驶、成本和收入|
|统一经营数据池|统一语义、周期、维度、指标和规划对比|
|经营分析|展示结果、偏差、原因、质量和下钻|
|模拟运行|推进统一时间并调用已有业务服务|

任何页面不得直接把经营目标和业务数组拼成分析结果；任何指标也不得反向修改规划或业务事实。
