# 前端体验规则

**适用范围：所有编码任务。** 即使任务描述不包含「前端」二字，只要代码修改会影响页面展示（新增页面、新增字段、新增按钮、修改表格、修改详情面板等），就必须遵守本规则。

## 1. 产品定位

前端是 Robotaxi 运营平台，不是营销页。

界面第一目标是让运营人员高效理解对象、单据、状态和操作闭环。

## 2. 空间密度

B 端运营系统以有效业务空间为核心。

表格、中控台、地图、详情、功能内容应获得最大可用面积。

导航、边栏、标题、说明和装饰区域应保持克制，在可读和可操作前提下尽量少占空间。

区域分隔优先使用细线、浅背景和紧凑间距，不依赖大面积空白制造层次。

## 3. 字体层级

默认字体层级：

|层级|用途|
|---|---|
|`12px`|表格内容、详情字段、筛选项、说明文字、状态值、输入内容|
|`13px`|一级 / 二级 / 三级菜单、按钮文字、重要字段标签|
|`14px`|模块标题、详情标题、表格区标题|
|`16px`|产品级标题上限，仅用于全局品牌或极少数全局页面标题|

普通业务页面禁止使用 `18px+` 大标题。

新增页面应向菜单尺度靠拢，避免标题占用过多内容空间。

## 4. 菜单规则

菜单应支持：

- 层级结构；
- 展开和收缩；
- 子级存在标识；
- 当前选中态；
- 父级高亮；
- 切换其他一级菜单时，未选中的展开组应收缩。

平台名称可作为回到运营中控台的入口。

## 5. 业务页面布局

运营中控台可以采用地图 + 详情的特殊布局。

其他业务对象、单据、策略、执行记录页面默认采用紧凑型业务列表页布局：

1. 页面标题区：显示模块名称和简短说明，保持紧凑。
2. 状态分类区：按核心状态提供 `全部 + 各状态` 快捷分类。
3. 搜索过滤区：按关键字段筛选记录。
4. 全局功能区：放对整个列表或筛选结果生效的操作。
5. 表格内容区：展示业务记录，字段名和值优先中文。
6. 单行固定操作列：表格最右侧固定显示单条记录操作。
7. 辅助信息区：展示事件、日志、校验结果等。
8. 底部信息区：展示图例、分页、筛选结果、统计或模块说明。

## 6. 筛选与状态分类

搜索过滤区必须形成闭环：

- 筛选字段；
- `查询`；
- `重置`。

状态分类区与搜索过滤区必须联动。

点击 `全部` 或某个状态分类时，应以状态分类为主重新计算列表，并清空临时筛选条件，避免旧筛选锁定数据。

菜单首次进入页面默认展示全部数据；非首次返回页面可以恢复该页面上次状态。

## 7. 表格操作

全局操作和单行操作必须分开。

全局操作放功能区。

单条记录操作放表格右侧固定列。

单行操作按钮必须统一样式和结构，不允许不同表格混用随意按钮、纯文本状态或未结构化操作。

单行操作只显示当前记录在当前状态下可执行的动作。

当前记录无状态变更动作但可查看对象时，统一显示 `查看详情`。

## 8. 详情栏

右侧详情栏必须跟随当前菜单和当前选中记录。

不得在非中控台页面继续显示地图详情。

详情栏应支持折叠和展开。

`查看详情` 应选中当前记录；如果右侧详情栏隐藏，则同时展开详情栏。

详情内容较多时，应使用分组、折叠或 Tab，不把所有字段平铺成冗长文本。

## 9. 工作台状态保留

随着页面增多，前端应支持业务页面状态保留。

可保留的页面状态包括：

- 状态分类；
- 搜索条件；
- 当前分页；
- 当前选中记录；
- Detail 展示状态。

工作台切换不应明显占用主业务区域。

如采用 Tab 或 Workspace，必须保持紧凑，并优先服务运营效率和空间利用。

工作台状态应结构化维护，避免把页面切换、筛选、分页、选中记录和详情展示状态散落在不同临时变量中。

后续新增业务对象、单据、策略或执行记录页面时，应默认接入统一工作台状态结构。

工作台页签视觉必须与运营后台整体风格一致：

- 页签区域只能单行展示，溢出时横向滚动，不得换行压缩核心业务区；
- 页签之间应有自然间距，避免过度拥挤；
- 当前选中页签应沿用系统主色、浅底色和轻边框表达，不使用突兀大面积强调；
- 页签区域应弱化边界，不额外制造厚重分割层；
- 中间业务区、表格区和右侧详情区的顶部与底部边界必须保持对齐；
- 新增工作台相关布局时必须同步检查高度计算，避免上下边界不平行。

## 10. 前端验证

前端功能完成后必须模拟真实运营人员操作路径。

至少覆盖本轮新增或修改的：

- 菜单进入；
- 状态分类；
- 查询和重置；
- 按钮点击；
- 表格选择；
- 详情展示；
- 关键状态变化；
- 刷新后的运行态持久化。
