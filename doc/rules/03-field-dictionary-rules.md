# 字段字典规则

## 1. 统一字段字典

统一字段字典位置：

`doc/common/field-dictionary.md`

所有业务对象、单据、策略、执行记录、状态、枚举、前端展示字段必须以统一字段字典为准。

业务目录下不再维护独立字段字典，只保留迁移说明或引用入口。

## 2. 字段定义原则

同一业务含义只能有一个稳定英文字段名。

字段字典至少包含：

- 英文字段名；
- 中文显示名；
- 字段性质；
- 字段含义；
- 必要时包含枚举取值和中文值。

## 3. 新增或修改字段

以下变化必须同步字段字典：

- 新增业务对象；
- 新增单据；
- 新增策略或执行记录；
- 新增字段；
- 字段含义变化；
- 状态或枚举变化；
- 前端显示名称变化。

## 4. 中文展示规则

前端表格、详情栏、筛选项、状态分类、操作上下文必须优先显示中文。

不得向运营人员直接暴露英文枚举值或内部字段名。

如发现前端出现英文代码值，优先检查：

1. 字段字典是否缺失；
2. 前端字段映射是否缺失；
3. 枚举中文值是否缺失；
4. 当前对象是否使用了兼容旧字段。

## 5. 兼容字段

兼容字段只用于旧数据或旧前端展示过渡。

新增代码不得优先依赖兼容字段。

发现兼容字段与新字段语义重复时，应在本轮分析中指出，并按用户确认逐步收敛。

## 6. 位置结构化表达

涉及位置的对象必须优先使用统一结构化表达。

Robotaxi、任务、行驶记录、服务订单等对象展示位置时，应尽量解释：

- Cell；
- ServiceArea；
- RoadSegment；
- Road；
- Place；
- Route；
- 当前业务上下文。

详情区默认展示摘要，完整上下文可折叠。
